# 앱 버전 관리

import { Callout } from 'nextra/components'

<Callout type="info">
  앱 버전 관리는 **사용자 경험의 일관성**과 **보안 패치 적용**을 보장하기 위한 핵심 운영 전략입니다. 강제 업데이트와 점진적 롤아웃을 조합하여 안정적으로 새 버전을 배포합니다.
</Callout>

## 버전 정책

### 버전 체계

```
major.minor.patch
  │      │     └─ 버그 수정, 핫픽스
  │      └─────── 새 기능 추가, 기존 기능 개선
  └────────────── 대규모 변경, 호환성 깨지는 변경
```

| 정책 | 내용 |
|------|------|
| **버전 형식** | `major.minor.patch` (예: 1.2.3) |
| **하위 호환** | 최신 버전 기준 하위 2개 minor 버전까지 지원 (예: 1.4.x 출시 시 1.2.x 이상 지원) |
| **보안 패치** | 보안 취약점 발견 시 **강제 업데이트** 대상 |
| **API 호환** | 서버 API는 최소 지원 버전까지 하위 호환 유지 |

## 강제 업데이트

### 서버 API

```
GET /api/config/app-version

Response:
{
  "ios": {
    "latestVersion": "1.3.0",
    "minimumVersion": "1.1.0",
    "forceUpdate": false,
    "updateUrl": "https://apps.apple.com/app/blockpick/id..."
  },
  "android": {
    "latestVersion": "1.3.0",
    "minimumVersion": "1.1.0",
    "forceUpdate": false,
    "updateUrl": "https://play.google.com/store/apps/details?id=..."
  },
  "maintenance": {
    "enabled": false,
    "message": "",
    "estimatedEndTime": null
  }
}
```

### 앱 시작 시 체크 로직

```
앱 시작
  └→ /api/config/app-version 호출
       ├→ 현재 버전 < minimumVersion → 강제 업데이트 다이얼로그 (닫기 불가)
       ├→ 현재 버전 < latestVersion → 권장 업데이트 다이얼로그 (닫기 가능)
       ├→ maintenance.enabled = true → 점검 화면 표시
       └→ 정상 → 앱 진입
```

### 강제 vs 권장 업데이트

| 유형 | 조건 | UI 동작 | 사용 시점 |
|------|------|---------|-----------|
| **강제 업데이트** | `현재 버전 < minimumVersion` | 전체 화면 다이얼로그, 닫기 불가, 스토어 이동만 가능 | 보안 취약점, 치명적 버그, API 호환성 깨짐 |
| **권장 업데이트** | `현재 버전 < latestVersion` | 바텀시트 다이얼로그, "나중에" 버튼으로 닫기 가능 | 새 기능 추가, UX 개선, 성능 향상 |

<Callout type="warning">
  강제 업데이트는 유저 이탈을 유발할 수 있으므로, **보안/치명적 이슈에만** 사용합니다. 가능하면 권장 업데이트로 유도합니다.
</Callout>

## 점진적 롤아웃

### Firebase Remote Config 기반 기능 플래그

| 플래그 | 타입 | 기본값 | 설명 |
|--------|------|--------|------|
| `feature_prime_game` | boolean | false | PRIME 역경매 게임 활성화 |
| `feature_mini_games_v2` | boolean | false | 미니게임 v2 UI 활성화 |
| `feature_social_share` | boolean | true | 소셜 공유 기능 |
| `max_daily_game_count` | number | 1 | DAILY 게임 일일 참여 제한 |
| `ad_reward_points` | number | 5 | 광고 시청 보상 포인트 |

### 롤아웃 단계

```
Phase 1: 내부 테스트 (QA 팀)
  └→ Phase 2: 10% 유저 (얼리 어답터 세그먼트)
       └→ Phase 3: 50% 유저 (일반 확대)
            └→ Phase 4: 100% 유저 (전체 공개)
```

| 단계 | 대상 | 기간 | 모니터링 |
|------|------|------|----------|
| 내부 테스트 | QA 팀 + 내부 직원 | 1~2일 | 크래시 리포트, 기능 정상 동작 |
| 10% 롤아웃 | 랜덤 10% 유저 | 2~3일 | 에러율, 성능 지표, 유저 피드백 |
| 50% 롤아웃 | 랜덤 50% 유저 | 2~3일 | 비즈니스 지표 (전환율, 매출), A/B 비교 |
| 100% 공개 | 전체 유저 | - | 최종 지표 확인 후 플래그 제거 |

<Callout type="info">
  각 단계에서 **에러율 1% 이상 증가** 또는 **크래시율 0.5% 초과** 시 즉시 롤백(플래그 off)합니다.
</Callout>

## 버전별 지원 정책

| 상태 | 조건 | 서비스 |
|------|------|--------|
| **최신** | `latestVersion` | 모든 기능 이용 가능 |
| **지원** | `minimumVersion` 이상 | 핵심 기능 이용 가능, 일부 신규 기능 제한 |
| **지원 종료** | `minimumVersion` 미만 | 앱 진입 불가, 강제 업데이트 필요 |

### 버전 지원 종료 프로세스

1. **2주 전**: 권장 업데이트 다이얼로그에 "지원 종료 예정" 문구 추가
2. **1주 전**: 푸시 알림 발송 ("앱 업데이트가 필요합니다")
3. **종료일**: `minimumVersion` 갱신 → 강제 업데이트 적용
