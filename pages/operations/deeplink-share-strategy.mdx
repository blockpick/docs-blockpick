## URL 스킴 설계

### 앱 내부 딥링크 (커스텀 스킴)

앱이 설치된 환경에서 직접 특정 화면을 여는 내부 링크입니다.

| 화면 | 스킴 | 파라미터 |
|------|------|----------|
| 홈 | `blockpick://home` | - |
| DAILY 게임 (오늘) | `blockpick://game/daily/today` | - |
| 특정 게임 상세 | `blockpick://game/{gameId}` | `gameId: string` |
| 게임 결과 | `blockpick://game/{gameId}/result` | `gameId: string` |
| PRIME 역경매 | `blockpick://game/prime/{gameId}` | `gameId: string` |
| VIBE 게임 | `blockpick://game/vibe/{gameId}` | `gameId: string` |
| 소원 상세 | `blockpick://wish/{wishId}` | `wishId: string` |
| 소원 등록 | `blockpick://wish/new` | - |
| 마이페이지 | `blockpick://mypage` | - |
| 당첨 상품 상세 | `blockpick://mypage/prizes/{prizeId}` | `prizeId: string` |
| 지갑/포인트 | `blockpick://mypage/wallet` | - |
| 출석체크 | `blockpick://attendance` | - |
| 리워드 광고 | `blockpick://rewards/ad` | - |
| 미니게임 | `blockpick://minigame` | - |
| 미니게임 상세 | `blockpick://minigame/{gameId}` | `gameId: string` |

### 유니버설 링크 (웹 URL 기반)

SNS 공유, 외부 링크, 브라우저 환경에서 사용하는 링크입니다. 앱이 설치된 경우 앱으로 직접 이동하고, 미설치 시 웹 랜딩 페이지로 fallback합니다.

| 콘텐츠 | URL 구조 |
|--------|----------|
| 소원 공유 | `https://blockpick.app/wish/{wishId}` |
| 게임 공유 | `https://blockpick.app/game/{gameId}` |
| 당첨 공유 | `https://blockpick.app/winner/{winnerId}` |
| 파트너 게임 (비회원) | `https://blockpick.app/partner/{gameId}` |
| 이벤트 랜딩 | `https://blockpick.app/event/{eventId}` |
| 초대 링크 | `https://blockpick.app/invite/{referralCode}` |

## Deferred Deep Link (지연 딥링크)

앱 미설치 유저가 공유 링크를 클릭했을 때, 설치 후 원래 목적지로 이동하는 흐름입니다. 소원 공유 링크를 통한 신규 유저 유입 시 가장 중요한 시나리오입니다.

### 전체 플로우

```
[공유 링크 클릭]
      |
      ↓
앱 설치 여부 확인
      |
  ┌───┴────┐
설치됨    미설치
  |         |
  ↓         ↓
앱 오픈   웹 랜딩 페이지 (blockpick.app/wish/{wishId})
  |         |
  |         ↓
  |    "앱에서 보기" CTA 버튼
  |         |
  |         ↓
  |    앱스토어 / 구글플레이로 이동
  |         |
  |         ↓
  |    앱 설치 완료
  |         |
  └────┬────┘
       ↓
[최초 앱 실행 시 딥링크 파라미터 복원]
       |
       ↓
목적지 화면으로 이동 (wish/{wishId})
```

### 딥링크 파라미터 보존 방법

| 방법 | 설명 | 적용 조건 |
|------|------|----------|
| **클립보드 방식** | 링크 클릭 시 딥링크 파라미터를 클립보드에 저장, 앱 최초 실행 시 읽기 | 간단하나 iOS 14+ 클립보드 접근 팝업 이슈 있음 |
| **서버 토큰 방식** | 서버에 딥링크 파라미터를 임시 저장 후 설치 완료 앱에서 조회 | 안정적, 서버 구현 필요 |
| **브랜치(Branch.io) 등 SDK** | 전문 딥링크 솔루션 사용 | 비용 발생, 가장 안정적 |

<Callout type="warning">
  구현 방식은 기술 아키텍트와 협의가 필요합니다. 기획 요구사항은 "앱 설치 후 원래 목적지로 이동"이며, 방법은 개발팀이 결정합니다.
</Callout>

### 웹 랜딩 페이지 요구사항

앱 미설치 유저가 처음 보는 페이지입니다. 설치 전환율을 높이는 것이 핵심 목표입니다.

| 구성 요소 | 내용 |
|-----------|------|
| 상품 이미지 | 소원/게임 대상 상품 대표 이미지 (크게) |
| 상품명 및 가격 | 명확한 상품 정보 |
| 참여 현황 | 현재 참여자 수, 남은 시간 |
| "지금 참여하기" CTA | 앱스토어/플레이스토어로 이동 |
| 서비스 소개 | 블록픽이 무엇인지 3줄 요약 |
| 플랫폼 뱃지 | App Store / Google Play 뱃지 |

## OG 메타 태그 설계

SNS(카카오톡, 인스타그램, X 등) 공유 시 미리보기에 표시되는 메타 데이터입니다.

### 소원 공유 OG

```html
<!-- 소원 공유 시 -->
<meta property="og:title" content="{상품명}을 소원해요 | 블록픽" />
<meta property="og:description" content="10원으로 {상품명}에 도전해보세요! 블록체인이 보장하는 공정한 추첨 🎯" />
<meta property="og:image" content="{상품 대표 이미지 URL}" />
<meta property="og:url" content="https://blockpick.app/wish/{wishId}" />
<meta property="og:type" content="website" />
```

### 게임 공유 OG

```html
<!-- 게임 공유 시 -->
<meta property="og:title" content="{상품명} 게임 진행 중 | 블록픽" />
<meta property="og:description" content="지금 {N}명이 참여 중! 블록픽 판에서 공정하게 추첨해요 ⚡" />
<meta property="og:image" content="{게임 대표 이미지 URL}" />
<meta property="og:url" content="https://blockpick.app/game/{gameId}" />
```

### 당첨 공유 OG

```html
<!-- 당첨 공유 시 -->
<meta property="og:title" content="블록픽에서 {상품명} 당첨됐어요! 🎉" />
<meta property="og:description" content="블록체인이 보장하는 공정한 추첨, 나도 도전해볼까요?" />
<meta property="og:image" content="{당첨 공유 카드 이미지 URL}" />
<meta property="og:url" content="https://blockpick.app/winner/{winnerId}" />
```

## 공유 카드 디자인 전략

소원 공유는 바이럴의 핵심입니다. 공유 카드가 SNS 피드에서 눈에 띄어야 클릭과 설치로 이어집니다.

### 소원 공유 카드

**목적**: 내가 원하는 상품을 자랑하며 공감과 참여를 유도

| 구성 요소 | 디자인 방향 |
|-----------|------------|
| 상품 이미지 | 전체 카드의 60~70% 면적, 고화질 |
| 상품명 | 큼직하게, 1줄 처리 (넘치면 말줄임) |
| 가격 | 서브 텍스트로 표시 |
| "내가 소원한 상품이에요" | 등록자 맥락 제공 문구 |
| 참여 CTA | "10원으로 같이 도전해요!" |
| 블록픽 로고 | 우하단 워터마크 |
| 배경 | 브랜드 그라디언트 또는 상품 이미지 블러 |

**카드 규격**
- 기본: 1200 × 630px (OG 이미지 표준)
- 인스타 스토리용: 1080 × 1920px (별도 생성)

### 당첨 공유 카드

**목적**: 당첨 사실을 자랑하며 신뢰도와 바이럴 유도

| 구성 요소 | 디자인 방향 |
|-----------|------------|
| "당첨!" 타이포 | 황금색, 임팩트 있게 |
| 당첨 상품 이미지 | 중앙 배치 |
| 당첨자 닉네임 | "OO님이 당첨됐어요!" |
| 블록체인 인증 배지 | "온체인 공정 추첨 인증" 뱃지 |
| 참여 유도 CTA | "나도 도전하기" 버튼 |
| 블록픽 로고 | 하단 |

<Callout type="info">
  당첨 공유 카드는 유저가 SNS에 직접 올리고 싶은 욕구를 자극해야 합니다. "내가 이걸 얻었다"는 자랑 욕구 + "블록체인으로 공정하게 뽑혔다"는 신뢰감이 핵심 메시지입니다.
</Callout>

### 공유 카드 생성 방법

| 방법 | 장단점 |
|------|--------|
| 서버 사이드 이미지 생성 (예: Sharp, Canvas) | 안정적, SEO 친화적 / 서버 구현 필요 |
| 앱 내 Flutter 위젯 → 이미지 저장 | 앱에서 즉시 생성 가능 / OG 이미지로 활용 어려움 |
| 정적 템플릿 + 동적 텍스트 오버레이 | 빠른 구현 / 디자인 자유도 낮음 |

## 공유 시나리오별 UX 플로우

### 소원 공유 플로우

```
소원 상세 화면
  → "공유하기" 버튼 탭
  → 공유 카드 미리보기 표시
  → 공유 채널 선택 (카카오톡 / 인스타그램 / 링크 복사)
  → 채널별 공유 실행
  → 공유 완료 → 소정의 포인트 보상 (옵션)
```

### 당첨 공유 플로우

```
당첨 결과 화면
  → 자동으로 당첨 공유 카드 표시 (풀스크린 모달)
  → "자랑하기" 버튼 탭
  → 공유 채널 선택
  → 채널별 공유 실행
```

### 초대 링크 공유 플로우

```
마이페이지 → "친구 초대"
  → 나만의 초대 코드가 포함된 링크 생성
  → https://blockpick.app/invite/{referralCode}
  → 공유 → 친구 가입 시 양방향 포인트 보상
```

## 카카오톡 공유 특별 전략

국내 사용자 특성상 카카오톡 공유가 가장 높은 클릭률을 보일 것으로 예상됩니다.

| 구성 요소 | 카카오톡 메시지 타입 | 내용 |
|-----------|---------------------|------|
| 피드 메시지 | 상품 이미지 + 제목 + 설명 + 버튼 2개 | "앱에서 보기" / "공유하기" |
| 버튼 1 | 앱 열기 딥링크 | `blockpick://wish/{wishId}` |
| 버튼 2 | 웹 랜딩 페이지 | `https://blockpick.app/wish/{wishId}` |

## 성공 지표 (KPI)

| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| 소원 공유율 | 게임 참여자의 20% 이상 | 공유 버튼 클릭 수 / 참여자 수 |
| 공유 → 앱 설치 전환율 | 15% 이상 | 공유 링크 클릭 → 앱 설치 완료 |
| 공유 → 게임 참여 전환율 | 30% 이상 | 딥링크 통해 진입한 유저의 게임 참여율 |
| 당첨 공유율 | 당첨자의 40% 이상 | 당첨 카드 공유 완료 수 / 당첨자 수 |
| Deferred Deep Link 목적지 이동 성공률 | 80% 이상 | 설치 후 목적지 화면 정상 진입 비율 |
