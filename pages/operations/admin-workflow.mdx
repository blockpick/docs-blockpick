# 어드민 운영 워크플로우

import { Callout } from 'nextra/components'

<Callout type="info">
  어드민은 블록픽 서비스의 운영 컨트롤 타워입니다. 게임 등록부터 당첨자 처리, CS 대응, 긴급 조치까지 모든 운영 행위가 이곳에서 이루어집니다.
</Callout>

## 전체 운영 화면 구성

| 화면 ID | 화면명 | 담당 업무 |
|---------|--------|----------|
| ADM-01 | 소원 관리 | 소원 검수(승인/반려), 부적절 소원 관리 |
| ADM-02 | 공동판 관리 | 판 생성/종료/모니터링, 소원 배치 현황 |
| ADM-03 | 업체 소원 등록 | 업체 소원 대리 등록, 광고 패키지 설정 |
| ADM-04 | 업체 광고 대시보드 | 업체별 노출/참여/클릭 통계, 잔액 관리 |
| ADM-05 | 드롭쉬핑 주문 관리 | 당첨 상품 구매/발송/추적 관리 |
| ADM-06 | PRIME 역경매 관리 | 역경매 상품 등록, 진행 현황, 결과 관리 |
| ADM-07 | 게임 관리 | 게임 CRUD, 스케줄링, 결과 발표 |
| ADM-08 | 사용자 관리 | 유저 조회, 제재, 포인트 조정 |
| ADM-09 | CS 관리 | 문의 접수/처리/완료 관리 |
| ADM-10 | 긴급 대응 | 게임 중단, 긴급 환불, 부정행위 제재 |

## 1. 게임 등록 → 스케줄링 → 결과 발표 전체 흐름

### 1.1 게임 등록 플로우 (ADM-07)

```
[게임 등록 시작]
      |
      ↓
Step 1: 게임 기본 정보 입력
  - 게임 타입 선택 (DAILY / VIBE / PRIME / PARTNER)
  - 게임 제목, 설명
  - 참여비 설정
  - 최대 참여 인원 설정
      |
      ↓
Step 2: 상품 등록
  - 상품명, 설명, 가격, 이미지 업로드
  - 당첨 상품 수량 및 등급 설정 (1등/2등/...)
  - 즉시 당첨 상품 여부 (PARTNER 게임)
      |
      ↓
Step 3: 스케줄 설정
  - 게임 시작 시각 (datetime picker)
  - 게임 종료 시각
  - 결과 발표 시각 (종료 후 자동 or 수동)
      |
      ↓
Step 4: 블록체인 설정 (DAILY/VIBE)
  - 온체인 기록 여부 확인
  - 스마트 컨트랙트 파라미터 검토
      |
      ↓
Step 5: 미리보기 및 검토
  - 앱 화면 미리보기
  - 최종 확인
      |
      ↓
[SCHEDULED 상태로 저장]
```

### 1.2 게임 상태 전이

| 상태 | 설명 | 전이 조건 |
|------|------|----------|
| `SCHEDULED` | 등록 완료, 시작 대기 | 등록 완료 즉시 |
| `ACTIVE` | 게임 진행 중 | 시작 시각 도달 (자동) |
| `ENDED` | 종료, 결과 처리 중 | 종료 시각 도달 (자동) |
| `RESULT_PUBLISHED` | 결과 발표 완료 | 운영자 결과 발표 액션 또는 자동 |
| `CANCELLED` | 게임 취소 | 운영자 수동 취소 |

<Callout type="warning">
  `ACTIVE` → `CANCELLED` 전이는 참여자 전원 자동 환불을 트리거합니다. 반드시 환불 처리 확인 후 상태 변경해야 합니다.
</Callout>

### 1.3 결과 발표 플로우

```
[게임 ENDED 상태]
      |
      ↓
당첨자 추첨 실행
  - DAILY/VIBE: 블록체인 기반 랜덤 추첨 (온체인 기록)
  - PRIME: 가장 낮고 유일한 입찰가 판정
  - PARTNER: 즉시 당첨 판정 (게임 참여 시 실시간)
      |
      ↓
당첨자 목록 확인 (운영자 검토)
  - 부정 참여 여부 확인 (동일 기기/계정 중복 등)
  - 이상 징후 없을 시 승인
      |
      ↓
결과 발표 (RESULT_PUBLISHED)
  - 당첨자 푸시 알림 자동 발송
  - 앱 게임 결과 화면 업데이트
  - 온체인 트랜잭션 해시 공개
      |
      ↓
당첨 상품 처리 (→ 섹션 2)
```

## 2. 당첨자 처리 워크플로우

### 2.1 기프티쇼 자동 발송 (디지털 상품)

```
[당첨자 확정]
      |
      ↓
상품 유형 판별: 디지털 쿠폰 여부 확인
      |
      ↓
기프티쇼 API 자동 호출 (백엔드)
  - 쿠폰 발급 요청
  - giftshowCouponUrl 수신
      |
      ↓
당첨자 앱 내 쿠폰 URL 전달
  - 마이페이지 > 당첨 상품 목록에 노출
  - 당첨 푸시 알림 발송 (쿠폰 링크 포함)
      |
      ↓
수령 확인
  - 쿠폰 열람 여부 추적
  - 7일 미수령 시 리마인더 푸시 발송
```

**자동 발송 실패 시 대응**

| 실패 원인 | 대응 방법 |
|----------|----------|
| 기프티쇼 API 오류 | 30분 간격 최대 3회 자동 재시도 |
| 3회 재시도 실패 | 운영자 수동 발급 알림 발생 |
| 재고 소진 | 동일 가치 대체 상품으로 교체 후 운영자 승인 |

### 2.2 실물 배송 처리 (드롭쉬핑, ADM-05)

```
[당첨자 확정]
      |
      ↓
배송지 정보 수집
  - 앱 내 배송지 입력 화면 알림 발송
  - 수령 기한: 당첨 후 72시간 이내 입력
      |
      ↓
ADM-05 주문 관리 화면에 주문 생성
  - 상품 구매처 URL 확인 (소원 등록 URL 기반)
  - 최저가 확인 (가격 변동 체크)
      |
      ↓
상품 구매 실행
  - 운영자가 구매처에서 직접 구매 (드롭쉬핑)
  - 주문 번호 ADM-05에 기록
      |
      ↓
발송 처리
  - 운송장 번호 입력
  - 택배사 선택
  - 발송 완료 → 당첨자 앱 내 배송 추적 제공
      |
      ↓
배송 완료 확인
  - 배송 완료 상태 업데이트
  - 완료 푸시 알림 발송
```

**실물 배송 상태 전이**

| 상태 | 설명 |
|------|------|
| `ADDRESS_PENDING` | 배송지 입력 대기 (72시간) |
| `ADDRESS_TIMEOUT` | 배송지 미입력 기한 초과 → 운영자 확인 필요 |
| `PURCHASING` | 운영자 상품 구매 진행 중 |
| `SHIPPING` | 발송 완료, 배송 중 |
| `DELIVERED` | 배송 완료 |
| `CS_OPEN` | 배송 관련 문의 접수됨 |

### 2.3 당첨 상품 유형별 처리 정책

| 상품 유형 | 처리 방식 | 처리 기한 |
|----------|----------|----------|
| 모바일 기프티쇼 쿠폰 | 자동 발송 | 즉시 (결과 발표 후 10분 이내) |
| 실물 상품 (소원 드롭쉬핑) | 수동 구매 + 발송 | 배송지 수집 후 3 영업일 이내 |
| 실물 상품 (업체 직배송) | 업체에 당첨자 정보 전달 | 결과 발표 후 1 영업일 이내 |
| 포인트 보상 | 자동 지급 | 즉시 |

## 3. CS 처리 워크플로우 (ADM-09)

### 3.1 문의 접수 채널

| 채널 | 설명 |
|------|------|

### 3.2 문의 처리 플로우

```
[문의 접수]
      |
      ↓
자동 분류 (카테고리 태깅)
  - 게임/당첨 관련
  - 포인트/결제 관련
  - 배송 관련
  - 계정/인증 관련
  - 부정행위 신고
  - 기타
      |
      ↓
담당자 배정 (또는 자동 응답)
  - 자동 응답 가능 여부 판단 (FAQ 매칭)
  - 수동 처리 필요 시 담당자 배정
      |
      ↓
처리 및 대응
  - 답변 작성 (템플릿 활용)
  - 필요 시 포인트 조정, 재발급, 재배송 실행
      |
      ↓
완료 처리
  - 문의 상태 CLOSED로 변경
  - 만족도 조사 발송 (옵션)
```

### 3.3 SLA (처리 기준 시간)

| 문의 유형 | 목표 응답 시간 | 목표 해결 시간 |
|----------|--------------|--------------|
| 당첨 관련 긴급 | 2시간 이내 | 4시간 이내 |
| 배송 관련 | 4시간 이내 | 2 영업일 이내 |
| 결제/포인트 관련 | 4시간 이내 | 1 영업일 이내 |
| 계정/기타 | 24시간 이내 | 3 영업일 이내 |

### 3.4 CS 처리 권한 레벨

| 레벨 | 가능한 조치 |
|------|------------|
| **L1 (일반 상담원)** | 답변 작성, 정보 조회, 표준 처리 |
| **L2 (시니어 상담원)** | 소액 포인트 조정 (5,000P 이하), 쿠폰 재발급 |
| **L3 (운영 관리자)** | 전액 환불, 게임 취소, 계정 제재/해제 |

## 5. 긴급 대응 절차 (ADM-10)

### 5.1 게임 중단 (Emergency Stop)

**발동 조건**

- 블록체인 트랜잭션 이상 감지
- 어뷰징/봇 참여 대규모 감지
- 서버/인프라 장애
- 상품 재고/가격 오류 확인

**처리 절차**

```
긴급 중단 버튼 클릭 (ADM-10)
      |
      ↓
중단 사유 입력 (필수)
      |
      ↓
게임 상태 → CANCELLED
      |
      ↓ (자동 트리거)
참여자 전원 참여비 자동 환불
      |
      ↓
공지 발송
  - 앱 푸시 알림: "시스템 점검으로 게임이 취소됐습니다. 참여비는 자동 환불됩니다."
  - 앱 내 공지 팝업 표시
      |
      ↓
사후 처리
  - 중단 원인 분석 및 보고서 작성
  - 재개 시 유저 사과 포인트 지급 여부 결정
```

<Callout type="warning">
  게임 중단 후 환불까지 최대 10분 이내 처리를 목표로 합니다. 환불 미처리 건이 발생할 경우 L3 운영 관리자가 수동으로 일괄 처리합니다.
</Callout>

### 5.2 긴급 환불

**발동 조건**

- 결제 오류로 인한 과다 차감
- 게임 오류로 인한 비정상 참여비 차감
- 운영 실수로 인한 게임 조건 오류

**처리 절차**

| 단계 | 내용 |
|------|------|
| 1. 대상자 조회 | 영향 받은 유저 목록 추출 (거래 이력 기반) |
| 2. 환불 금액 산정 | 과다 차감 금액 계산 |
| 3. 승인 | L3 운영 관리자 승인 (단건 5만원 초과 시 2인 승인) |
| 4. 환불 실행 | 포인트 잔액 일괄 복구 |
| 5. 알림 발송 | 개별 유저에게 환불 완료 푸시 알림 |
| 6. 기록 | 환불 사유, 금액, 처리자를 감사 로그에 기록 |

### 5.3 부정행위 제재

**탐지 기준**

| 유형 | 탐지 방법 | 기준 |
|------|----------|------|
| 다계정 참여 | 동일 기기 ID 복수 계정 | 1기기 1계정 원칙 위반 |
| 봇 참여 | 비정상적 참여 패턴 (시간 간격, IP 등) | 자동화 도구 사용 의심 |
| 결제 어뷰징 | 결제 후 즉시 환불 반복 | 3회 이상 반복 시 |
| 소원 어뷰징 | 타인 소원 무더기 신고 | 24시간 내 10건 이상 신고 |

**제재 단계**

| 단계 | 제재 내용 | 기간 |
|------|----------|------|
| 1차 경고 | 경고 메시지 발송 | - |
| 2차 제한 | 게임 참여 일시 정지 | 7일 |
| 3차 정지 | 계정 이용 정지 | 30일 |
| 영구 제재 | 계정 영구 정지 + 보유 포인트 몰수 | 영구 |

<Callout type="warning">
  당첨 후 부정행위 확인 시: 당첨 취소 → 참여비 몰수 → 차순위 당첨자 재선정. 모든 제재는 감사 로그에 기록하며, 제재 사유와 증거를 명확히 남겨야 합니다.
</Callout>

### 5.4 긴급 대응 에스컬레이션 체계

```
[이상 감지]
      |
      ↓
L1 운영자: 이상 여부 1차 판단 (15분 이내)
      |
      ↓ (심각도 HIGH 이상)
L2 시니어: 영향 범위 파악 + 임시 조치 (30분 이내)
      |
      ↓ (서비스 전체 영향)
L3 관리자: 게임 중단 여부 결정 + 공지 승인 (1시간 이내)
      |
      ↓ (외부 노출 / 법적 이슈)
대표자 보고
```

## 운영 일정 가이드
