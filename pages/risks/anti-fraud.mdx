# 부정행위 방지 전략

블록픽은 돈이 걸린 상품 추첨 서비스입니다. 부정행위는 공정성 훼손, 매출 손실, 법적 리스크로 직결되므로 런칭 전부터 체계적인 방지 전략이 필요합니다.

import { Callout } from 'nextra/components'

<Callout type="warning">
  부정행위 방지는 **서비스 신뢰의 핵심**입니다. 한 건의 어뷰징 사례가 커뮤니티에 공유되면 전체 유저 이탈로 이어질 수 있습니다. P0 항목은 반드시 런칭 전에 구현해야 합니다.
</Callout>

---

## 1. 부정행위 유형별 분석

### 1-1. 다중 계정 어뷰징

한 사람이 여러 계정을 생성하여 당첨 확률을 높이거나 무료 포인트를 반복 획득하는 행위입니다.

**공격 시나리오**

| 시나리오 | 설명 |
|----------|------|
| 다중 기기 | 여러 스마트폰으로 계정 다수 생성 |
| 앱 초기화 | 앱 삭제/재설치 반복으로 신규 혜택 수령 |
| 가상 번호 | 대량 인증용 가상 전화번호 사용 |
| 가족/지인 동원 | 타인 명의 계정으로 본인이 운영 |

**탐지 방법**

| 방법 | 설명 |
|------|------|
| Device Fingerprint | 기기 고유 식별자(ADID/IDFV + 하드웨어 해시) 수집 |
| IP 클러스터링 | 동일 IP 대역에서 다수 계정 생성 탐지 |
| 전화번호 중복 검증 | 가상 번호 DB 대조, 통신사 실명 인증 연동 |
| 행동 패턴 유사도 | 게임 참여 시간, 선호 게임, 소비 패턴이 유사한 계정 그룹 탐지 |

**방지 정책**

- 동일 Device Fingerprint로 최대 1개 계정만 허용
- 신규 가입 시 SMS 실명 인증 필수 (CI/DI 기반)
- 가상 번호 사업자 DB 블랙리스트 유지 관리
- 동일 IP에서 24시간 내 3개 이상 가입 시 자동 차단 + 수동 검토 큐

**대응**

- 중복 계정 발견 시: 후순위 계정 자동 정지, 획득 포인트 회수
- 이미 당첨된 경우: 당첨 무효 처리, 차순위 유저에게 재추첨

### 1-2. 봇/매크로

미니게임 자동 플레이, 자동 추첨 참여 등 프로그래밍된 봇을 이용한 부정행위입니다.

**공격 시나리오**

| 시나리오 | 설명 |
|----------|------|
| 미니게임 매크로 | ADB/Accessibility API로 미니게임 자동 클리어하여 포인트 획득 |
| 자동 참여 봇 | DAILY/VIBE 게임에 자동 참여 스크립트 실행 |
| 에뮬레이터 팜 | PC 에뮬레이터 다수 실행으로 대량 계정 운영 |

**탐지 방법**

| 방법 | 설명 |
|------|------|
| 입력 패턴 분석 | 터치 좌표 분산, 입력 간격 표준편차 측정 — 기계적 패턴 탐지 |
| 반응 시간 이상 | 인간 반응 한계(~200ms) 이하의 연속 반응 탐지 |
| 정확도 이상 | 미니게임 연속 만점 또는 비정상 정확도 패턴 |
| 세션 연속성 | 24시간+ 무중단 세션 탐지 |

**방지 정책**

- 미니게임 결과는 서버에서 최종 검증 (클라이언트 결과 신뢰 금지)
- 의심 세션에 CAPTCHA (reCAPTCHA v3 + 풀백 v2) 강제
- 에뮬레이터/루팅 탐지 (SafetyNet/Play Integrity API, DeviceCheck)
- 미니게임 포인트 획득에 일일 횟수 제한 적용

**대응**

- 봇 탐지 시: 해당 세션 포인트 전액 회수, 계정 경고
- 반복 적발 시: 영구 차단 + Device Fingerprint 블랙리스트 등록

### 1-3. 광고 리워드 어뷰징

AdMob 리워드 광고를 비정상적으로 반복 시청하여 포인트를 부정 획득하는 행위입니다.

**공격 시나리오**

| 시나리오 | 설명 |
|----------|------|
| VPN 우회 | VPN으로 국가 변경하여 광고 인벤토리 회피 후 반복 시청 |
| 에뮬레이터 시청 | 에뮬레이터에서 자동화된 광고 시청 |
| 빠른 스킵 | 네트워크 조작으로 광고 시청 완료를 위조 |

**탐지 방법**

| 방법 | 설명 |
|------|------|
| AdMob SSVC | Server-Side Verification Callback으로 실제 시청 여부 검증 |
| 에뮬레이터 탐지 | SafetyNet Attestation / App Attest |
| VPN 탐지 | IP 평판 DB 대조 (MaxMind, IPQualityScore) |
| 시청 패턴 | 일일 시청 빈도, 시청 간격 분석 |

**방지 정책**

- AdMob SSVC 필수 적용 — 서버에서 콜백 검증 후에만 포인트 지급
- `idempotencyKey`로 동일 광고 시청에 대한 중복 지급 방지
- 일일 리워드 광고 시청 한도: 최대 **10회** (서버 제어)
- 에뮬레이터/VPN 환경에서는 리워드 광고 비활성화

<Callout type="info">
  AdMob SSVC는 이미 통합 설계에 포함되어 있습니다. 상세는 [AdMob 통합](/integrations/admob)을 참조하세요.
</Callout>

**대응**

- SSVC 검증 실패 시: 포인트 미지급 (자동)
- 패턴 어뷰징 탐지 시: 광고 리워드 기능 일시 차단 (24시간)

### 1-4. 블록체인 트랜잭션 조작

온체인에 기록된 게임 데이터(좌표, 참여 기록)를 조작하려는 시도입니다.

**공격 시나리오**

| 시나리오 | 설명 |
|----------|------|
| 좌표 위변조 | 당첨 좌표를 사전에 알아내거나 변경 시도 |
| 이중 참여 | 동일 게임에 여러 번 참여하여 확률 증가 시도 |
| 프론트러닝 | 블록체인 mempool 모니터링으로 당첨 좌표 선점 시도 |
| 리플레이 공격 | 이전 트랜잭션을 재전송하여 중복 참여 |

**탐지 방법**

| 방법 | 설명 |
|------|------|
| 온체인 무결성 | AES-256-CBC 암호화된 좌표는 게임 종료 전 복호화 불가 |
| 컨트랙트 검증 | 스마트 컨트랙트에서 참여자별 1회 참여만 허용 |
| 서버 이중 검증 | 서버에서도 참여 기록을 별도 관리하여 컨트랙트와 교차 검증 |
| Nonce 관리 | 트랜잭션 Nonce로 리플레이 공격 차단 |

**방지 정책**

- 좌표 암호화 키는 서버에서만 관리 (AES-256-CBC)
- 게임 시작 시 암호화된 좌표를 온체인에 기록 → 게임 종료 시 복호화 키 공개
- 스마트 컨트랙트 `require` 조건으로 이중 참여 원천 차단
- 유저 지갑은 자동 생성/관리 — 프라이빗 키 직접 접근 불가

<Callout type="info">
  블록체인 보안 아키텍처 상세는 [블록체인 보안](/blockchain/security) 및 [Ronin 통합](/integrations/ronin)을 참조하세요.
</Callout>

**대응**

- 조작 시도 탐지 시: 해당 게임 참여 무효, 계정 즉시 정지
- 컨트랙트 취약점 발견 시: 비상 중지(Pause) 기능 실행 후 패치

### 1-5. 결제 사기 (환불 어뷰징)

인앱 결제로 포인트를 충전한 뒤 사용하고, 앱스토어를 통해 환불을 요청하는 행위입니다.

**공격 시나리오**

| 시나리오 | 설명 |
|----------|------|
| 충전→사용→환불 | 포인트 충전 후 게임에 사용, 이후 스토어에 환불 요청 |
| 카드 도용 | 도용 카드로 대량 충전 후 빠르게 소진 |
| 가족 공유 악용 | 가족 결제 승인 취소 기능으로 환불 |

**탐지 방법**

| 방법 | 설명 |
|------|------|
| 환불 빈도 모니터링 | 유저별 환불 횟수/비율 추적 |
| 마이너스 잔액 추적 | 환불 시점에 이미 포인트를 사용한 경우 마이너스 잔액 기록 |
| 사용 속도 분석 | 충전 후 비정상적으로 빠른 소비 패턴 감지 |
| 스토어 알림 연동 | Apple/Google 서버 알림(Server Notification)으로 환불 실시간 감지 |

**방지 정책**

- 환불 발생 시 해당 포인트 즉시 차감 (마이너스 허용)
- 마이너스 잔액 상태에서는 게임 참여/포인트 사용 차단
- 환불 2회 이상 시 해당 결제 수단 차단 + 계정 검토 플래그
- 대량 충전 직후 즉시 사용 패턴 탐지 시 출금/참여 지연(쿨다운 24시간)

**대응**

- 마이너스 잔액: 다음 충전 시 자동 상계 처리
- 악의적 환불 반복: 계정 영구 차단, 당첨 이력 무효화
- 카드 도용 의심: 결제 즉시 동결, 본인 인증 재요청

### 1-6. 소원 시스템 어뷰징

소원(Wish) 시스템에서 허위 상품을 등록하거나, 자기 공감/댓글 스팸으로 노출도를 조작하는 행위입니다.

**공격 시나리오**

| 시나리오 | 설명 |
|----------|------|
| 허위 상품 등록 | 존재하지 않는 상품 URL, 품절 상품, 비정상 가격 상품 등록 |
| 자기 공감 | 다중 계정으로 본인 소원에 공감 수 조작 |
| 댓글 스팸 | 자동화된 댓글로 소원 노출도 인위적 상승 |
| 추천 조작 | 소문내기(VIBE) 게임에서 자기 소원만 반복 공유 |

**탐지 방법**

| 방법 | 설명 |
|------|------|
| URL 유효성 검증 | 상품 URL 실시간 크롤링 — 404, 품절, 가격 이상 탐지 |
| 공감 중복 검증 | 동일 IP/디바이스에서의 중복 공감 차단 |
| 댓글 스팸 필터 | 유사도 분석, 댓글 빈도 제한, 금칙어 필터 |
| 공감 패턴 분석 | 짧은 시간 내 급증하는 공감에 대한 이상 탐지 |

**방지 정책**

- 소원 등록 시 URL 자동 검증 (상품 존재 여부, 가격 범위 체크)
- 공감은 계정당 1회 + IP/디바이스 중복 차단
- 댓글은 계정당 동일 소원에 최대 3개, 10초 간격 제한
- 소문내기 참여 시 동일 소원 연속 공유 제한 (일 3회)

**대응**

- 허위 상품 탐지: 소원 비공개 전환, 등록자 경고
- 공감 조작: 부정 공감 수 제거, 관련 계정 경고
- 스팸 댓글: 자동 삭제, 반복 시 댓글 기능 일시 차단

### 1-7. 공유/추천 어뷰징

추천인 코드를 악용하여 추천 보상을 부정 획득하는 행위입니다.

**공격 시나리오**

| 시나리오 | 설명 |
|----------|------|
| 자기 추천 | 다중 계정으로 본인 추천 코드를 입력하여 보상 획득 |
| 추천 팜 | 대량의 가짜 계정 생성 후 추천 보상 수확 |
| 커뮤니티 악용 | 불특정 다수에게 추천 코드를 스팸 배포 |

**탐지 방법**

| 방법 | 설명 |
|------|------|
| 추천인 검증 | 추천인 계정의 활동 이력, 가입 기간 검증 |
| IP/디바이스 대조 | 추천인과 피추천인의 IP, Device Fingerprint 비교 |
| 추천 속도 분석 | 단시간 내 대량 추천 발생 시 자동 플래그 |
| 피추천인 활동 검증 | 추천 보상은 피추천인의 첫 게임 참여 후 지급 (가입만으로 불가) |

**방지 정책**

- 동일 IP/디바이스에서 생성된 계정 간 추천 차단
- 추천 보상 지급 조건: 피추천인 실명 인증 완료 + 첫 게임 참여
- 계정당 추천 보상 수령 한도: 월 **30건**
- 추천인과 피추천인의 Device Fingerprint 유사도 검사

**대응**

- 자기 추천 탐지: 추천 보상 회수, 관련 계정 모두 경고
- 추천 팜 탐지: 추천인 계정 영구 차단, 발급된 보상 전액 회수

## 2. 제재 프로세스

<Callout type="warning">
  제재는 **자동 탐지 → 1차 검토(시스템) → 2차 검토(운영자)** 프로세스를 거칩니다. 영구 차단은 반드시 운영자 수동 승인이 필요합니다.
</Callout>

### 단계별 제재

| 단계 | 조치 | 조건 | 자동/수동 |
|------|------|------|-----------|
| **1차 경고** | 경고 알림 발송, 어뷰징 기록 | 최초 탐지 | 자동 |
| **2차 포인트 동결** | 포인트 사용/출금 72시간 동결 | 경고 후 재적발 | 자동 |
| **3차 일시 정지** | 계정 7일 정지, 진행 중 게임 참여 취소 | 동결 후 재적발 | 자동 + 운영자 확인 |
| **4차 영구 차단** | 계정 영구 정지, 잔여 포인트 동결, Device Fingerprint 블랙리스트 | 3차 정지 후 재적발 또는 심각한 부정행위 | 운영자 수동 승인 필수 |

### 이의 제기

| 항목 | 내용 |
|------|------|
| **접수 채널** | 앱 내 고객센터, 이메일 |
| **처리 기한** | 접수 후 영업일 기준 3일 이내 |
| **검토 절차** | 로그 분석 → 오탐 여부 확인 → 결과 통보 |
| **오탐 시** | 즉시 제재 해제, 보상 포인트 지급 |

## 3. 모니터링 & 대시보드

### 실시간 이상 탐지

| 모니터링 항목 | 임계값 | 알림 |
|---------------|--------|------|
| 동일 IP 다중 가입 | 24시간 내 3건 이상 | Slack + 어드민 알림 |
