# 보안

블록체인 관련 보안 정책입니다.

## 키 관리 (NFR-007)

| 항목 | 내용 |
|------|------|
| **암호화** | AES-256 |
| **저장소** | Flutter Secure Storage (Keychain/Keystore) |
| **니모닉** | BIP39 표준 |
| **핵심 원칙** | 프라이빗 키는 디바이스 외부로 전송하지 않음 |

## 게임 결과 무결성 (NFR-010)

| 항목 | 내용 |
|------|------|
| **방식** | 좌표 암호화 후 블록체인 기록 |
| **보장** | 게임 종료 전 복호화 불가 |
| **검증** | 트랜잭션 해시로 누구나 검증 가능 |
| **핵심 원칙** | 백엔드 운영자도 결과를 미리 알 수 없음 |

## 보안 고려사항

- 프라이빗 키는 디바이스 Keychain/Keystore에만 저장합니다
- 서버로 프라이빗 키를 전송하는 것은 금지됩니다
- 스마트 컨트랙트에 대한 보안 감사(Audit)가 필요합니다
- 사용자가 직접 키를 관리하지 않습니다 (자동 생성/보관)

## 리스크

| 리스크 | 확률 | 영향 | 완화 방안 |
|--------|------|------|-----------|
| 스마트 컨트랙트 취약점 | 낮음 | 높음 | 보안 감사, 단계적 배포, 비상 중지 기능 |
| 지갑 키 분실 (앱 삭제) | 중 | 중 | 키 백업 방안 검토 (서버 에스크로 또는 시드 문구 안내) |
| 블록체인 네트워크 장애 | 중 | 높음 | 오프체인 Fallback, 복구 후 자동 동기화 |
