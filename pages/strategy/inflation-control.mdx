# 포인트 인플레이션 관리 전략

블록픽 포인트 경제의 **장기 안정성**을 유지하기 위한 공급 제어, 소각 메커니즘, 모니터링 체계를 정의합니다.

import { Callout } from 'nextra/components'

<Callout type="warning">
  포인트 인플레이션은 서비스 초기에는 눈에 띄지 않지만, **유저 수가 10만 이상** 되면 급격히 문제가 됩니다. 사전 설계 없이 런칭하면 사후 조정이 매우 어렵습니다.
</Callout>

## 1. 인플레이션 리스크 분석

### 1-1. 인플레이션이란?

포인트 인플레이션 = **총 유통 포인트가 총 소각량보다 빠르게 증가**하여, 포인트 1P의 체감 가치가 하락하는 현상.

```
  [정상 상태]                              [인플레이션 상태]
  발행 ≈ 소각                              발행 >> 소각

  포인트 가치: 안정                         포인트 가치: 하락
  유저 체감: "100P면 게임 한 판"             유저 체감: "100P가 별로 안 아까움"
  과금 동기: 유지                           과금 동기: 하락
```

### 1-2. 무료 포인트 과잉 공급 시나리오

| 시나리오 | 발생 조건 | 예상 영향 |
|---------|---------|----------|
| **미니게임 어뷰징** | 봇/매크로로 미니게임 대량 플레이 | 일일 상한 무력화, 무료P 폭증 |
| **이벤트 남발** | 마케팅팀의 무분별한 보너스 이벤트 | 단기 유입↑, 장기 가치 하락 |
| **출석 보상 누적** | 장기 유저 대량 출석 보상 누적 | 소비 없는 포인트 잉여 |
| **공감 어뷰징** | 다계정으로 소원 공감 남발 | 일일 100P 무한 생성 |
| **위로 포인트 누적** | 대량 참여 + 전원 미당첨 | 미당첨 위로P가 순발행량 증가 |

### 1-3. 포인트 가치 하락이 서비스에 미치는 영향

```
  [포인트 가치 하락]
        │
        ├──→ 과금 동기 감소: "굳이 충전 안 해도 무료로 충분하네"
        │         → 매출 하락
        │
        ├──→ 참여비 무의미화: "100P 써도 아깝지 않음"
        │         → 게임 참여의 스릴 감소
        │         → 리텐션 하락
        │
        ├──→ 상품 원가 부담 증가: 참여비 대비 상품 원가 비율 악화
        │         → 수익성 악화
        │
        └──→ 밸런스 조정 시 반발: "갑자기 보상 줄이면 안 되지!"
                  → 유저 이탈
```

<Callout type="error">
  **최악의 시나리오**: 포인트 가치 하락 → 과금 감소 → 상품 품질 하락 → 유저 이탈 → 더 많은 무료 이벤트로 유저 붙잡기 → 더 심한 인플레이션. **악순환의 시작**입니다.
</Callout>

## 2. 공급 제어 (Faucet)

### 2-1. 채널별 일일 공급 상한

| 채널 | 일일 상한 (1인) | 일일 상한 근거 | 조정 가능 범위 |
|------|---------------|--------------|--------------|
| 출석체크 (기본) | 10P | 고정 | 5~15P |
| 출석 광고 부스터 | +10P | 1회/일 | 변동 없음 |
| 미니게임 클리어 | 50P | 일 최대 캡 | 30~80P |
| 미니게임 광고 부스터 | 50P | 5회/일 x 2배 | 3~7회/일 |
| 소원 공감 | 100P | 20회/일 x 5P | 10~30회/일 |
| 홈 광고 보상 | 25P | 5회/일 x 5P | 3~7회/일 |
| 미당첨 위로 (DAILY) | 3P/회 | 참여 횟수 비례 | 1~5P/회 |
| 미당첨 위로 (WISH) | 2P/회 | 참여 횟수 비례 | 1~3P/회 |
| **무과금 일일 최대** | **~248P** | | |

### 2-2. 무과금 일일 최대 248P의 근거

```
  248P/일의 설계 의도:

  ① 게임 참여 전환
     248P ÷ 100P(DAILY) = 2.48회/일 추가 참여 가능
     248P ÷ 10P(WISH) = 24.8회/일 소문내기 가능
     → "무과금으로도 적당히 즐길 수 있지만, 더 하려면 과금"

  ② 월간 환산
     248P × 28일 = 6,944P/월
     → DAILY 69회 or WISH 694회
     → "충분히 체험 가능하지만, 헤비 유저에겐 부족"

  ③ 과금 대비 비율
     최소 충전 패키지: 5,000P (₩5,000)
     무과금 월간 획득: 6,944P
     → 무과금:과금 = 약 1.4:1 (1회 충전 기준)
     → "소액 충전 1회 ≈ 무과금 1달" → 충전 가치 체감
```

<Callout type="info">
  **248P는 "충분히 맛보기 + 부족함 체감"의 균형점**입니다. 이 수치를 늘리면 과금 동기가 약해지고, 줄이면 무과금 유저 이탈이 늘어납니다. 론칭 후 데이터 기반으로 ±20% 범위에서 조정합니다.
</Callout>

### 2-3. 조정 기준 (언제 상한을 변경하는가)

| 지표 | 상한 증가 조건 | 상한 감소 조건 |
|------|-------------|-------------|
| 무과금 D7 리텐션 | 20% 미만 | 50% 이상 |
| 무과금→과금 전환율 | 5% 이상 (이미 충분) | 1% 미만 |
| 일일 순발행량/DAU | - | 300P+ (과잉 공급) |
| 포인트 소비율 | 60% 미만 (쌓이기만 함) | - |

### 2-4. 이벤트 시 추가 공급 가이드라인

| 이벤트 등급 | 추가 공급 한도 | 기간 제한 | 승인 절차 |
|-----------|-------------|---------|----------|
| **S등급** (론칭, 연말) | 일일 상한 +100% | 최대 7일 | CEO 승인 |
| **A등급** (시즌 이벤트) | 일일 상한 +50% | 최대 5일 | 사업팀장 승인 |
| **B등급** (일반 이벤트) | 일일 상한 +30% | 최대 3일 | 운영팀장 승인 |
| **C등급** (소규모) | 일일 상한 +10% | 최대 1일 | 운영팀 자율 |

<Callout type="warning">
  **이벤트 쿨다운 규칙**: S/A등급 이벤트 종료 후 최소 14일간 동등 이상 등급 이벤트 금지. 연속 이벤트는 유저의 "이벤트가 항상 있으니 평소엔 안 해도 돼" 학습을 유발합니다.
</Callout>

## 3. 소각 메커니즘 (Sink)

### 3-1. 소각 채널 분류

```
  ┌─────────────────────────────────────────────────┐
  │              포인트 소각 (Sink)                    │
  ├─────────────────────────────────────────────────┤
  │                                                 │
  │  [자연 소각]              [강제 소각]              │
  │  유저 행동에 의한 소각     시스템에 의한 소각        │
  │                                                 │
  │  • 게임 참여비           • 90일 무료 포인트 소멸    │
  │  • PRIME 입찰금          • 쿠폰 30일 소멸          │
  │  • 프리미엄 기능 (향후)                            │
  │                                                 │
  └─────────────────────────────────────────────────┘
```

### 3-2. 게임 참여비 소각

게임 참여비는 블록픽의 **1차 소각 메커니즘**이자 **1차 수익원**입니다.

| 게임 | 참여비 | 일평균 참여 (DAU 10만 기준) | 일일 소각량 |
|------|--------|--------------------------|-----------|
| DAILY (유료 참여) | 100P | 30,000회 | 3,000,000P |
| WISH (소문내기) | 10P | 80,000회 | 800,000P |
| PRIME (입찰) | 평균 300P | 5,000회 | 1,500,000P |
| **합계** | | **115,000회** | **5,300,000P** |

### 3-3. 90일 무료 포인트 소멸

| 소멸 대상 | 유효기간 | 소멸 알림 | 예상 소멸률 |
|----------|---------|----------|-----------|
| 무료 획득 포인트 전체 | 90일 | 만료 7일 전 + 3일 전 | 15~25% |
| 쿠폰 포인트 | 30일 | 만료 3일 전 | 30~40% |
| 이벤트 보너스 포인트 | 이벤트별 상이 | 이벤트 종료 3일 전 | 20~30% |

**소멸이 소각에 기여하는 비율 추정**:

```
  DAU 100,000 기준 월간 소멸 시뮬레이션:

  무과금 유저 (70%): 70,000명
  월간 획득: 70,000 × 6,944P = 486,080,000P
  소멸률 20%: 486,080,000 × 0.20 = 97,216,000P 소멸/월

  → 월간 게임 참여 소각 (5,300,000P × 30): 159,000,000P
  → 월간 소멸 소각: 97,216,000P
  → 소멸은 전체 소각의 약 38% 기여
```

### 3-4. 프리미엄 기능 포인트 소비 (Phase 2)

향후 추가할 포인트 소비처로, **게임 참여 외의 소각 채널**을 확보합니다.

| 프리미엄 기능 | 소비 포인트 | 도입 시기 | 소각 기여도 |
|-------------|-----------|---------|-----------|
| 프로필 꾸미기 (아바타/배지) | 50~500P | Phase 2 | 낮음 |
| 소원 상단 노출 부스트 | 100P | Phase 2 | 중 |
| DAILY 참여 순서 우선권 | 200P | Phase 3 | 중 |
| 미니게임 특별 모드 해금 | 30P | Phase 2 | 낮음 |
| 당첨 확률 분석 리포트 | 500P | Phase 3 | 낮음 |

<Callout type="info">
  프리미엄 기능은 **당첨 확률에 영향을 주지 않는** 범위에서만 설계합니다. 블록체인 공정 추첨의 핵심 가치를 훼손하면 안 됩니다.
</Callout>

## 5. 긴급 조정 프로토콜

### 5-1. 인플레이션 감지 시 대응 단계

```
  [Level 1: 주의] ──→ [Level 2: 경고] ──→ [Level 3: 위험] ──→ [Level 4: 비상]
   발행/소각 1.3+      발행/소각 1.5+      발행/소각 2.0+      유저 체감 변화
   3일 연속             3일 연속             1일 이상            과금 급감
       │                    │                    │                   │
       ▼                    ▼                    ▼                   ▼
   모니터링 강화         공급 조정 시작        긴급 소각 가동       구조적 리밸런싱
```

#### Level 1: 주의 (모니터링 강화)

| 조치 | 내용 | 담당 |
|------|------|------|
| 원인 분석 | 어떤 채널에서 과잉 발행 중인지 확인 | 데이터팀 |
| 이벤트 점검 | 진행 중 이벤트의 보상 규모 검증 | 운영팀 |
| 일일 리포트 | 채널별 발행/소각 상세 리포트 시작 | 자동 |

#### Level 2: 경고 (공급 조정)

| 조치 | 내용 | 영향도 |
|------|------|--------|
| 미니게임 일일 상한 20% 감소 | 50P → 40P | 무과금 유저 일일 획득 10P 감소 |
| 공감 보상 20% 감소 | 5P → 4P | 무과금 유저 일일 획득 20P 감소 |
| 진행 중 이벤트 보상 30% 감소 | 이벤트별 상이 | 이벤트 효과 감소 |
| 신규 이벤트 승인 동결 | B등급 이하 이벤트 금지 | 마케팅 활동 제한 |

#### Level 3: 위험 (긴급 소각)

| 조치 | 내용 | 영향도 |
|------|------|--------|
| 무료 공급 전 채널 50% 감소 | 248P → 124P | 무과금 유저 체감 큼 |
| 임시 포인트 소비 이벤트 | "포인트 2배 사용 기간" (소각 가속) | 소각량 일시 급증 |
| 90일 소멸 → 60일 단축 (신규 발행분) | 기발행분은 유지 | 장기 소각 가속 |
| 어뷰징 계정 일괄 정지 | 비정상 패턴 계정 | 공급 원인 차단 |

#### Level 4: 비상 (구조적 리밸런싱)

| 조치 | 내용 | 리스크 |
|------|------|--------|
| 게임 참여비 인상 | DAILY 100P → 150P 등 | 유저 반발 |
| 신규 소각 채널 긴급 도입 | 프리미엄 기능 조기 런칭 | 품질 리스크 |
| 포인트 리디노미네이션 | 전체 잔액 비율 조정 | 유저 혼란 (최후 수단) |

<Callout type="error">
  **Level 4는 최후 수단**입니다. 여기까지 오면 이미 유저 신뢰가 손상된 상태입니다. Level 1~2에서 조기에 대응하는 것이 핵심입니다.
</Callout>

### 5-2. 이벤트 보상 조정 가이드라인

모든 이벤트는 **사전 인플레이션 영향 평가(IIA)**를 통과해야 합니다.

| 평가 항목 | 기준 | 통과 조건 |
|----------|------|----------|
| 총 추가 발행량 | 이벤트 기간 × 대상 유저 × 보상 | 월간 총 소각량의 10% 이하 |
| 발행/소각 비율 변동 | 이벤트 기간 중 예상 비율 | 1.5 미만 |
| 소각 연동 여부 | 이벤트 보상에 소비 조건 부착 | 권장 (50% 이상 소비 조건) |
| 쿨다운 준수 | 이전 동등 이벤트 이후 경과일 | 14일 이상 |

**이벤트 보상 설계 공식**:

```
  추가 발행 한도 = 월간 평균 소각량 × 0.10 ÷ 이벤트 일수 ÷ 예상 참여자 수

  예시:
  월간 소각량: 200,000,000P
  이벤트 일수: 3일
  예상 참여자: 50,000명

  1인당 일일 추가 보상 한도 = 200,000,000 × 0.10 ÷ 3 ÷ 50,000 = 133P
```
