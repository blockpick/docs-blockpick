## 0. 확장 전략 개요

블록픽의 확장은 **코어 루프(추첨 → 당첨 → 공유) 강화**를 최우선으로 하되, 검증된 성장 지표를 기반으로 단계적으로 확장합니다.

```
수평 확장 (기능)          수직 확장 (가치 체인)
─────────────────         ──────────────────────
소셜 기능                  자체 상품 브랜드
크리에이터 이코노미    +    마켓플레이스
구독 모델                  기프트/선물
NFT 통합                   B2B 플랫폼

        지역 확장                 플랫폼 확장
        ──────────               ─────────────
        국내 → 일본      +       웹 버전
        → 동남아                 카카오 미니앱
                                 오프라인 연동
```

**확장 원칙**: MAU 지표 달성 후 순차 진행. 코어 루프를 희석하는 기능은 도입하지 않습니다.

## 1. 수평 확장: 기능 확장

### 1-1. 소셜 기능 강화

**도입 시기**: Phase 2 (MAU 2만+ 달성 후)

<Callout type="info">
  소셜 기능은 기존 바이럴 루프를 **앱 내 커뮤니티**로 내재화하여 리텐션을 높이는 것이 목적입니다.
</Callout>

| 기능 | 설명 | 기대 효과 | 우선순위 |
|---|---|---|---|
| 팔로우/팔로잉 | 유저 간 소원 구독 | D30 리텐션 +5%p | 높음 |
| 소원 피드 | 팔로우한 유저의 소원 타임라인 | 세션 시간 +2분 | 높음 |
| 소원 랭킹 | 공감 수 기준 주간/월간 랭킹 | 경쟁 심리 유발, 공유율 증가 | 중간 |
| 당첨 축하 기능 | 당첨자에게 이모지·댓글 | 커뮤니티 감성 강화 | 중간 |
| 소원 컬렉션 | 소원 카테고리별 큐레이션 | 신규 유저 소원 등록 진입장벽 낮춤 | 낮음 |

**소셜 피드 설계 원칙**

```
[피드 알고리즘]
1순위: 팔로잉한 유저의 소원 (최신 순)
2순위: 내 소원과 카테고리 유사 + 공감 급증
3순위: 지역 기반 인기 소원
4순위: 파트너 브랜드 소원 (광고 라벨 표시)
```

### 1-2. 크리에이터 이코노미

**도입 시기**: Phase 2 후반 (MAU 3만+ 달성 후)

**소원 큐레이터 프로그램**

| 등급 | 조건 | 혜택 |
|---|---|---|
| 일반 유저 | 가입 즉시 | 소원 등록, 공감, 참여 |
| 큐레이터 | 팔로워 500명+, 공감 합계 1,000+ | 큐레이터 뱃지, 우선 노출, 추가 포인트 |
| 파트너 큐레이터 | 팔로워 5,000명+, 월 활동 기준 | 수익 쉐어 (파트너 소원 유입 기준), 전용 채널 |
| 공식 인플루언서 | 블록픽 선정 | 전용 소원 개설 권한, 마케팅 협업 |

**크리에이터 수익 구조**

```
파트너 큐레이터가 공유한 소원 링크를 통해 유저 유입
    ↓
유입 유저가 포인트 충전
    ↓
충전액의 3% → 큐레이터 포인트로 지급
(월 최대 50만 포인트 = 50만 원 상당)
```

### 1-3. 구독 모델

**도입 시기**: Phase 3 (MAU 20만+ 달성 후)

<Callout type="warning">
  구독 모델은 무료 유저 경험을 저해하지 않아야 합니다. 구독 혜택은 **편의성과 양적 확대** 중심으로 설계하고, 공정성은 구독 여부와 무관하게 동일하게 보장합니다.
</Callout>

| 티어 | 월 요금 | 핵심 혜택 | 타겟 유저 |
|---|---|---|---|
| 블록픽 Plus | 2,900원 | 매일 추가 미니게임 3회, 포인트 소멸 60일 연장 | 미니게임 헤비 유저 |

```
무료 유저 → 미니게임 참여 제한 도달 → Plus 안내 팝업
충전 유저 → PRIME 참여 경쟁 심화 → Pro 업그레이드 유도
파워 유저 → 전용 혜택 가시화 → VIP 전환
```

**구독 목표 수익 (MAU 50만 기준)**

| 티어 | 전환율 | 구독자 수 | 월 수익 |
|---|---|---|---|
| Plus | 3% | 15,000명 | 4,350만 원 |
| Pro | 1% | 5,000명 | 3,950만 원 |
| VIP | 0.2% | 1,000명 | 1,990만 원 |
| **합계** | **4.2%** | **21,000명** | **약 1억 290만 원** |

### 1-4. NFT 통합

**도입 시기**: Phase 3 (Ronin 네트워크 활용, MAU 10만+ 후)

<Callout type="info">
  NFT는 투기 목적이 아닌 **당첨 인증 + 커뮤니티 정체성** 도구로 도입합니다. 기존 블록체인(Ronin) 인프라를 활용해 추가 비용을 최소화합니다.
</Callout>

| NFT 유형 | 발행 조건 | 혜택 | 희소성 |
|---|---|---|---|
| 당첨 인증 NFT | 게임 당첨 시 자동 발행 | 앱 프로필 표시, 커뮤니티 인증 | 당첨 게임별 1개 |
| 시즌 참여 NFT | 시즌 이벤트 완료 | 다음 시즌 혜택 +10% | 시즌별 한정 |
| 파트너 협업 NFT | 파트너 브랜드 소원 당첨 | 파트너사 실물 혜택 | 브랜드 협의 |
| 레전드 소원 NFT | 공감 1만 이상 소원 | 명예의 전당 등재 | 극희소 |

## 2. 수직 확장: 가치 체인 확장

### 2-1. 자체 상품 브랜드

**도입 시기**: Phase 3 (MAU 30만+ 달성 후)

**전략 방향**: 블록픽 전용 상품을 추첨 대상으로 제공 → 마진율 향상 + 브랜드 강화

| 카테고리 | 상품 예시 | 마진율 목표 | 소싱 방식 |
|---|---|---|---|
| 블록픽 굿즈 | 블록픽 로고 MD, 한정판 아이템 | 60%+ | 자체 제작 |
| PB 전자기기 | 블록픽 에디션 이어폰/스마트워치 | 30%+ | OEM 협력 |
| 디지털 상품 | 구독권, 포인트 팩 | 90%+ | 자체 발행 |

### 2-2. 당첨 상품 마켓플레이스 (2차 거래)

**도입 시기**: Phase 2 후반 (MAU 3만+ 달성 후)

**구조**: 당첨 상품을 수령하지 않고 앱 내에서 다른 유저에게 양도

```
당첨 발생
    ↓
수령 vs 마켓에 올리기 선택
    ↓
마켓에 올리면 즉시 포인트로 전환 (시세의 70~90%)
    ↓
구매자는 포인트 또는 현금으로 구매
    ↓
블록픽 수수료: 거래금액의 5%
```

**기대 효과**

| 효과 | 내용 |
|---|---|
| 당첨 후 이탈 방지 | 상품 불필요 유저도 포인트로 보상 → 이탈 없이 재참여 |
| 포인트 수요 증가 | 마켓 구매용 포인트 충전 유인 |
| 거래 수수료 수익 | 월 거래액 5억 원 × 5% = 월 2,500만 원 (MAU 50만 기준) |

### 2-3. 기프트/선물 기능

**도입 시기**: Phase 2 (시즌 이벤트 연동)

| 기능 | 설명 | 수익화 방식 |
|---|---|---|
| 소원 대신 이뤄주기 | 타인의 소원에 포인트 기부 | 기부 포인트 수수료 2% |
| 게임 참여권 선물 | 게임 참여 기회를 친구에게 선물 | 참여권 판매 마진 |
| 당첨 상품 선물 | 당첨 상품을 지인에게 배송 | 배송비 + 처리 수수료 |
| 생일 소원 알림 | 친구 생일에 소원 알림 | 기프트 콘텐츠 내 광고 |

## 4. 플랫폼 확장

### 4-1. 웹 버전 (PC 유저)

**도입 시기**: Phase 2 (MAU 2만+ 달성 후)

<Callout type="info">
  웹 버전은 앱의 **보완재**로 설계합니다. 소원 등록·공유·당첨 확인은 웹에서 가능하게 하되, 미니게임·PRIME은 앱 전용으로 앱 설치를 유도합니다.
</Callout>

| 기능 | 웹 버전 | 앱 전용 |
|---|---|---|
| 소원 등록/공감 | O | O |
| 당첨 결과 확인 | O | O |
| DAILY 게임 참여 | 제한적 (1일 1회) | 무제한 |
| 미니게임 | X (앱 유도) | O |
| PRIME 역경매 | X (앱 유도) | O |
| 포인트 충전 | O | O |

**웹 유입 → 앱 전환 전략**

```
웹에서 소원 공감
    ↓
"게임에 참여하려면 앱이 필요해요" 배너
    ↓
QR코드 / 앱 스토어 링크
    ↓
앱 설치 → 웹 세션 연속 (딥링크)
```

### 4-2. 카카오 미니앱

**도입 시기**: Phase 2 (카카오 공유 트래픽 1만 클릭/월 이상 시)

| 항목 | 내용 |
|---|---|
| 제공 기능 | 소원 공감, 당첨 결과 확인, 초대 코드 입력 |
| 목적 | 카카오 공유 링크 클릭 → 앱 설치 없이 즉시 참여 |
| 앱 전환 유도 | 참여 완료 후 "더 많은 게임은 앱에서" |
| 기대 전환율 | 미니앱 방문자의 25%가 앱 설치 |

### 4-3. 오프라인 연동

**도입 시기**: Phase 3 (파트너 10개+ 확보 후)

**QR 기반 오프라인 → 앱 연결**

| 시나리오 | 방식 | 파트너 유형 |
|---|---|---|
| 매장 방문 인증 | 매장 QR 스캔 → 방문 포인트 + 소원 참여권 | 카페, 음식점, 편의점 |
| 영수증 인증 | 영수증 QR → 구매 금액 비례 포인트 | 리테일, 마트 |
| 이벤트 현장 | 오프라인 이벤트 QR → 현장 전용 게임 | 팝업스토어, 페스티벌 |
| 상품 패키지 | 파트너 상품 패키지 내 QR | FMCG 브랜드 |

**오프라인 연동 수익 구조**

| 항목 | 단가 | 월 목표 수익 |
|---|---|---|
| QR 설치 파트너십 | 월 30만~100만 원/매장 | 3,000만 원 (100개 매장) |
| 영수증 포인트 수수료 | 포인트 발행액의 10% | 1,000만 원 |
| 현장 전용 게임 스폰서 | 이벤트당 500만~1,000만 원 | 2,000만 원 |

## 5. 확장별 리스크 & 의존성

### 5-1. 확장 로드맵 의존성 다이어그램

```
[코어 루프 안정화]
        │
        ├─→ [소셜 기능] ─→ [크리에이터 이코노미] ─→ [NFT 통합]
        │
        ├─→ [웹 버전] ─→ [카카오 미니앱]
        │
        ├─→ [마켓플레이스] ─→ [기프트 기능]
        │
```
