# 유저 세그먼트별 과금 전략

블록픽 유저를 과금 수준별로 세그먼트화하고, 각 세그먼트에 최적화된 전환 전략과 LTV 시뮬레이션을 정의합니다.

import { Callout } from 'nextra/components'

<Callout type="info">
  블록픽의 과금 모델은 "확률 구매"가 아니라 **"참여 기회 구매"**입니다. 과금은 당첨 확률이 아닌 참여 횟수를 늘리므로, 기존 가챠와는 과금 심리가 다릅니다. 이 차이를 세그먼트 전략에 반영합니다.
</Callout>

### 1-1. 과금 수준별 4단계

```
  ┌──────────────────────────────────────────────────────┐

  ├──────────────────────────────────────────────────────┤
  │                                                      │
  │                      ┌─────┐                         │
  │                      │Whale│  3~5%                    │
  │                      │월5만+│                         │
  │                    ┌─┴─────┴─┐                       │
  │                    │ Dolphin  │  10~15%               │
  │                    │ 월1~5만  │                       │
  │                  ┌─┴─────────┴─┐                     │
  │                  │   Minnow     │  30~40%             │
  │                  │  광고시청형   │                     │
  │                ┌─┴─────────────┴─┐                   │
  │                │    Free Rider    │  40~50%           │

  │                                                      │
```

### 1-2. 세그먼트 상세

| 세그먼트 | 정의 | 예상 비율 | 월 과금액 | 주요 행동 |
|---------|------|---------|----------|----------|
| **Free Rider** | 과금 0, 광고 시청도 최소 | 40~50% | ₩0 | 무료 참여 1회/일만, 낮은 체류 |
| **Minnow** | 과금 0, 광고 적극 시청 | 30~40% | ₩0 (광고 수익 기여) | 미니게임 + 광고로 포인트 확보, 높은 체류 |
| **Dolphin** | 소액 정기 과금 | 10~15% | ₩10,000~50,000 | 주 1~2회 충전, DAILY/WISH 적극 참여 |
| **Whale** | 중대과금 | 3~5% | ₩50,000~300,000 | 매주 충전, PRIME 적극 참여, 다수 게임 동시 |

## 2. 세그먼트별 전환 전략

### 2-1. Free Rider → Minnow (활성화 전환)

**목표**: 앱을 깔고 거의 안 쓰는 유저를 **매일 접속하는 유저**로 전환

```
  [Free Rider]                          [Minnow]
  "깔긴 했는데 잘 안 함"    ──→    "매일 접속해서 포인트 모으기"

  전환 트리거:
  ① 푸시 알림: "오늘 무료 참여 안 하셨어요! (매일 자정 리셋)"
  ② 미당첨 시: "미니게임으로 포인트 모아보세요 (무료)"
  ③ 소원 공감: "○○님이 내 소원에 공감했어요!"
```

| 전환 전술 | 타이밍 | 기대 전환율 |
|----------|--------|-----------|
| 재참여 푸시 (무료 참여 미사용) | 매일 18:00 | 8~12% |
| 소원 공감 알림 (소셜 트리거) | 공감 발생 시 | 15~20% |
| 7일 미접속 → 복귀 보너스 (+100P) | 복귀 시 | 5~8% |
| 인기 상품 알림 | 신규 상품 등록 시 | 3~5% |

### 2-2. Minnow → Dolphin (첫 과금 전환, F2P Conversion)

**목표**: 광고만 보던 유저가 **처음으로 돈을 쓰게** 만들기

<Callout type="warning">
  **첫 과금은 가장 높은 허들**입니다. 한 번 과금한 유저의 재과금률은 60~70%이지만, 첫 과금 전환율은 업계 평균 2~5%입니다. 블록픽은 "참여 기회 구매"의 가성비를 체감하게 하는 것이 핵심입니다.
</Callout>

```
  [Minnow의 일상]
  미니게임 10판 → 50P 획득 → DAILY 참여 불가 (100P 필요)
                                    │
                              ┌─────┴─────┐
                              ▼           ▼
                        [광고 시청]   [포인트 부족 팝업]
                        추가 2회      "5,000P 충전 시
                                      50회 참여 가능!
                                      첫 충전 30% 보너스"
                                           │
                                           ▼
                                    [첫 과금 결정]
```

**첫 과금 유도 전술**:

| 전술 | 설계 | 심리 원리 | 기대 전환율 |
|------|------|----------|-----------|
| **첫 충전 보너스** | 첫 충전 30% 추가 (5,000P → 6,500P) | 손실 회피 ("이 보너스 놓치면 손해") | +2~3% |
| **스타터 패키지** | ₩1,000에 1,500P (50% 할인) 한정 1회 | 미끼 효과 (저가 진입점) | +3~5% |
| **포인트 부족 시 넛지** | 참여하려는데 포인트 부족 → 충전 유도 팝업 | 근접 효과 ("거의 다 왔는데") | +1~2% |
| **한정 상품 FOMO** | "iPhone 한정 3개! 지금 참여하려면 충전" | 희소성 + FOMO | +2~4% |
| **연속 참여 위기** | "7일 연속 기록! 오늘 참여 안 하면 초기화" | 매몰 비용 | +1~3% |

**최적 첫 과금 가격대**:

| 충전 패키지 | 가격 | 포인트 | 보너스 | 게임 환산 | 타겟 |
|-----------|------|--------|--------|----------|------|
| **스타터** (첫 충전 한정) | ₩1,000 | 1,500P | +50% | DAILY 15회 | 최저 허들 |
| **라이트** | ₩5,000 | 6,500P | +30% (첫 충전) | DAILY 65회 | 표준 진입 |
| **스탠다드** | ₩10,000 | 12,000P | +20% | DAILY 120회 | 정기 과금 입문 |

### 2-3. Dolphin → 정기 과금 습관 형성 (Retention)

**목표**: 소액 과금 유저가 **매월 정기적으로 충전**하는 습관 형성

```
  [Dolphin의 과금 여정]

  월 1회 충전 (₩10,000)
        │
        ▼
  주간 미션 보상 ← "이번 주 3회 참여 시 보너스 500P"
        │
        ▼
  월간 패스 구매 ← "월 ₩9,900에 매일 보너스 포인트 + 전용 상품"
        │
        ▼
  정기 구독 전환 ← "자동 충전 설정 시 5% 추가 보너스"
```

| 습관 형성 전술 | 설계 | 목표 |
|-------------|------|------|
| **주간 미션 보상** | 주 3회+ 참여 시 보너스 100P | 주 단위 참여 습관 |
| **월간 패스** | ₩9,900/월 → 매일 200P + 전용 상품 접근 | 월 정기 과금 |
| **자동 충전 보너스** | 월 자동 충전 설정 시 5% 추가 | 이탈 방지 |
| **충전 누적 등급** | 3개월 연속 충전 → VIP 배지 + 특별 이벤트 | 장기 과금 |

### 2-4. Whale → 지속적 가치 제공 (VIP 관리)

**목표**: 고과금 유저에게 **지속적으로 과금할 이유**를 제공

| VIP 혜택 | 조건 | 내용 |
|---------|------|------|
| **VIP 전용 상품** | 월 5만원+ 충전 | 일반 유저 미공개 프리미엄 상품 |
| **PRIME 우선 알림** | 월 5만원+ 충전 | 신규 PRIME 상품 30분 선공개 |
| **위로 보상 추가** | 월 10만원+ 충전 | 미당첨 위로 포인트 2배 |
| **전용 고객 지원** | 월 20만원+ 충전 | 카카오톡 1:1 전담 CS |
| **당첨 확률 분석** | 월 5만원+ 충전 | 개인 입찰 패턴 분석 리포트 |

<Callout type="warning">
  **Whale 혜택은 "편의성과 독점 경험"**이지, **당첨 확률 우대가 아닙니다**. 블록체인 공정 추첨 원칙은 어떤 세그먼트에도 예외 없이 적용됩니다.
</Callout>

## 3. 과금 유도 트리거 상세 설계

### 3-1. 포인트 부족 시 충전 유도 UX

```
  [유저가 게임 참여 버튼 터치]
        │
        ▼
  [잔액 확인: 참여비 > 현재 잔액?]
        │
  ┌─────┴─────┐
  │  충분      │  부족
  ▼           ▼
 참여 진행    ┌─────────────────────────────┐
              │  포인트가 부족합니다          │
              │                             │
              │  현재 잔액: 45P              │
              │  필요 포인트: 100P           │
              │  부족분: 55P                 │
              │                             │
              │  ┌────────────┐ ┌────────┐  │
              │  │ 광고 시청    │ │ 충전하기 │  │
              │  │ (남은 2회)  │ │        │  │
              │  └────────────┘ └────────┘  │
              │                             │
              │  💡 5,000P 충전 시 50회 참여!  │
              └─────────────────────────────┘
```

**핵심**: 광고 시청 옵션을 먼저 보여줘서 "강제 과금" 느낌을 피하되, 충전의 가성비를 자연스럽게 노출합니다.

### 3-2. 한정 상품 출현 시 FOMO 활용

```
  [홈 피드에 한정 상품 등장]
        │
        ▼
  ┌─────────────────────────────────────┐
  │  🔥 한정 상품 등장!                   │
  │                                     │
  │  [iPhone 16 Pro] 3개 한정            │
  │  현재 참여자: 234명                   │
  │  마감까지: 02:34:56                  │
  │                                     │
  │  참여비: 100P                        │
  │  내 잔액: 12P                        │
  │                                     │
  │  ⚡ 지금 충전하면 첫 충전 보너스 30%!  │
  │  [바로 충전하기]                      │
  └─────────────────────────────────────┘
```

| FOMO 요소 | 효과 | 측정 지표 |
|----------|------|----------|
| 수량 한정 ("3개 한정") | 긴급성 부여 | 한정 상품 참여율 |
| 타이머 | 시간 압박 | 마감 1시간 내 참여 비율 |
| 실시간 참여자 수 | 경쟁 심리 | 참여자 증가 속도 |
| 잔액 부족 + 충전 넛지 | 과금 전환 | 충전 전환율 |

```

  "오늘 참여하면 +170P 연속 보너스"
        │
        ▼
  [잔액 부족]
        │
        ▼
  ┌─────────────────────────────────────┐
  │  ⚠️ 14일 연속 기록이 끊길 위기!       │
  │                                     │
  │  지금까지 누적 보너스: 170P           │
  │  다음 보너스 (28일): +500P           │

  │  오늘 참여를 놓치면 기록이 초기화됩니다  │
```

**전환 효과 예상**:

## 4. 세그먼트별 LTV 시뮬레이션

### 4-1. LTV 산출 모델

```
  LTV = ARPU × (1 / Churn Rate) × Gross Margin

  또는

  LTV = Σ (월 ARPU × 생존율(n)) for n = 1~36개월
```

### 4-2. 세그먼트별 핵심 지표

| 지표 | Free Rider | Minnow | Dolphin | Whale |
|------|-----------|--------|---------|-------|
| 월 ARPU | ₩0 | ₩150 | ₩25,000 | ₩120,000 |
| 월간 이탈률 | 30% | 15% | 8% | 5% |
| 평균 잔존 기간 | 3.3개월 | 6.7개월 | 12.5개월 | 20개월 |
| Gross Margin | - | 60% (광고) | 35% (상품 원가 차감) | 30% (상품 원가 차감) |

### 4-3. 세그먼트별 LTV 계산

| 세그먼트 | 월 ARPU | 평균 잔존 | Gross Margin | **LTV** |
|---------|---------|---------|-------------|---------|
| Free Rider | ₩0 | 3.3개월 | - | **₩0** |
| Minnow | ₩150 | 6.7개월 | 60% | **₩603** |
| Dolphin | ₩25,000 | 12.5개월 | 35% | **₩109,375** |
| Whale | ₩120,000 | 20개월 | 30% | **₩720,000** |

### 4-5. CAC(고객 획득 비용) 허용 범위

```
  허용 CAC = LTV × 목표 LTV/CAC 비율

  업계 기준 LTV/CAC 비율: 3:1 이상
  블록픽 목표: 4:1 (보수적)
```

| 세그먼트 | LTV | 허용 CAC (4:1) | 허용 CAC (3:1) | 획득 채널 |
|---------|-----|--------------|--------------|----------|
| Free Rider | ₩0 | ₩0 | ₩0 | 오가닉/바이럴만 |
| Minnow | ₩603 | ₩150 | ₩200 | 오가닉/바이럴 |
