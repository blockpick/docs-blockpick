# 포인트 선순환 구조 & 소비 동기 설계

블록픽 포인트가 "벌어서 쓰는" 단순 루프가 아니라, **쓸수록 더 쓰고 싶어지는** 자기 강화형 플라이휠(Flywheel)을 설계합니다.

import { Callout } from 'nextra/components'

<Callout type="info">
  이 문서는 [포인트 이코노미 & 리텐션 전략](/currency/point-economy)의 수치 체계를 기반으로, **"왜 유저가 계속 포인트를 소비하고 싶어하는가"**의 심리적 동기 구조를 다룹니다.
</Callout>

## 1. 선순환 플라이휠 모델 (Virtuous Cycle Flywheel)

### 1-1. 전체 루프 구조

```
          ┌──────────────────────────────────────────────────────┐
          │                                                      │
          ▼                                                      │
   ① 획득 (Earn)                                                │
   미니게임·광고·출석·공감                                        │
          │                                                      │
          ▼                                                      │
   ② 축적 (Accumulate)                                          │
   잔액 증가 → "이걸로 뭘 할 수 있지?"                            │
          │                                                      │
          ▼                                                      │
   ③ 소비 (Spend)                                               │
   DAILY·WISH·PRIME 게임 참여                                    │
          │                                                      │
          ▼                                                      │
   ④ 보상/경험 (Reward & Experience)                             │
   당첨 → 상품 수령 / 미당첨 → 위로P + 스릴                      │
          │                                                      │
          ▼                                                      │
   ⑤ 재획득 욕구 (Re-earn Desire)                                │
   "다시 해보고 싶다" / "아까워서 멈출 수 없다"                    │
          │                                                      │
          └──────────────────────────────────────────────────────┘
```

### 1-2. 각 단계별 심리적 트리거

| 단계 | 유저 심리 | 행동경제학 원리 | 블록픽 설계 |
|------|----------|----------------|------------|
| ① 획득 | "공짜로 벌 수 있다" | 제로 비용 효과 (Zero Price Effect) | 미니게임 무료, 광고 시청 보상 |
| ② 축적 | "이만큼 모았는데 안 쓰면 아깝다" | 매몰 비용 (Sunk Cost) | 잔액 표시 + 90일 소멸 카운트다운 |
| ③ 소비 | "이 가격에 이 상품을? 해볼 만하다" | 앵커링 (Anchoring) | 100P(≈100원)로 에어팟 도전 |
| ④ 보상 | "나도 당첨될 수 있다!" | 가능성 과대평가 (Probability Weighting) | 실시간 당첨 알림 + 후기 피드 |
| ⑤ 재욕구 | "한 번만 더..." | 가변 비율 강화 (Variable Ratio Reinforcement) | 매번 다른 결과 + 위로P 즉시 지급 |

<Callout type="warning">
  **핵심 설계 원칙**: 루프의 어느 단계에서든 유저가 이탈하지 않도록, 각 단계에 **다음 단계로 넘어가게 하는 넛지**가 존재해야 합니다.
</Callout>

### 1-3. 플라이휠 가속 조건

플라이휠이 돌수록 **관성이 붙어야** 합니다. 아래 3가지가 가속 조건입니다:

```
  [유저 수 증가]  ──→  [게임당 참여자 증가]  ──→  [당첨 빈도 증가]
        ↑                                              │
        │                                              ▼
  [바이럴 유입]  ◀──  [당첨 후기 증가]  ◀──  [소셜 프루프 강화]
```

| 가속 지표 | 측정 방법 | 목표 |
|----------|----------|------|
| 플라이휠 회전 속도 | 평균 획득→소비 전환 시간 | 24시간 이내 |
| 재참여율 | 소비 후 48시간 내 재소비 비율 | 60% 이상 |
| 바이럴 계수 | 당첨자 1명당 신규 유입 수 | 0.3명 이상 |

## 2. 게임별 소비 동기 설계

### 2-1. DAILY: "오늘 안 하면 놓치는" 일일 한정 스릴

```
                    ┌─────────────────────┐
                    │  매일 자정 리셋       │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │  무료 참여 1회 제공    │  ← "공짜인데 안 하면 손해"
                    └──────────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                ▼
        [당첨!]          [미당첨]          [추가 참여?]
        상품 수령        위로 3P           100P or 광고
              │                │                │
              ▼                ▼                ▼
        후기 작성        "내일은 될 거야"    "한 번만 더"
        +30~100P         연속 보너스 축적     FOMO 자극
```

**소비 동기 레이어**:

| 동기 레이어 | 심리 원리 | 구체적 설계 |
|------------|----------|------------|
| 1차: 무료 참여 | 손실 회피 ("안 하면 손해") | 매일 1회 무료 → 로그인 유도 |
| 2차: 연속 보너스 | 매몰 비용 ("3일 연속인데 끊을 순 없지") | 3일 +20P, 7일 +50P, 연속 기록 UI 강조 |
| 3차: 추가 참여 | 근접 효과 ("거의 당첨될 뻔했는데") | 미당첨 시 "다음 참여 할인" 팝업 |
| 4차: FOMO | 희소성 ("오늘 상품 마감까지 3시간") | 상품별 마감 타이머 + 잔여 수량 표시 |

### 2-2. WISH: "내 소원이 이뤄질 수 있다" 감정적 투자

WISH는 블록픽에서 **가장 감정적 투자가 깊은** 게임입니다. "내 소원"이라는 심리적 소유권(Endowment Effect)이 핵심입니다.

```
  [내가 소원 등록] ──→ [공감 수집] ──→ [게임 개설] ──→ [누군가 당첨!]
        │                  │                              │
        │                  │                              ▼
        │                  │                    [나도 1+1 혜택!]
        │                  ▼                              │
        │          "내 소원에 100명이                       ▼
        │           공감해줬어!"                     "내 소원 이뤄졌다!"
        ▼                                          소셜 공유 + 바이럴
  "내 소원이니까
   더 열심히 공유해야지"
```

**소비 동기 레이어**:

| 동기 레이어 | 심리 원리 | 구체적 설계 |
|------------|----------|------------|
| 1차: 소원 등록 | 감정적 소유권 (Endowment) | "내 소원"에 대한 애착 → 게임 개설까지 지속 관여 |
| 2차: 공감 활동 | 호혜성 (Reciprocity) | 남의 소원에 공감 → 남도 내 소원에 공감 기대 |
| 3차: 소문내기 참여 | 사회적 증거 (Social Proof) | "1,234명 참여 중" 실시간 표시 |
| 4차: 바이럴 공유 | 자기 표현 (Self-Expression) | "이 상품 갖고 싶은 사람?" → SNS 공유 |

<Callout type="info">
  WISH의 핵심은 **참여비(10P)가 아니라 감정적 투자**입니다. 소원 등록자는 게임이 성공하길 원하기 때문에 자발적으로 홍보합니다. 이는 마케팅 비용 0원의 바이럴 엔진입니다.
</Callout>

### 2-3. PRIME: "이 가격에 이 상품을?" 역경매 스릴

```
  [고가 상품 공개]
  "iPhone 16 Pro가 1P부터?"
        │
        ▼
  [입찰 금액 결정]        ← 앵커링: 정가 대비 극적 할인
  "500P면 되지 않을까?"
        │
        ▼
  [결과 확인]
        ├── 당첨 → "500원에 iPhone!!" (극적 체험)
        │
        └── 미당첨 → 입찰금 50% 환급
                     "250P 돌려받았으니 다시 해볼까?"
                              │
                              ▼
                     [재입찰 유도]
                     "이번엔 금액을 바꿔볼까?"
```

**소비 동기 레이어**:

| 동기 레이어 | 심리 원리 | 구체적 설계 |
|------------|----------|------------|
| 1차: 가격 갭 | 앵커링 (정가 vs 참여비) | "정가 1,890,000원 상품을 500P에 도전" |
| 2차: 안전망 | 손실 완화 (Loss Cushion) | 미당첨 시 50% 환급 → 체감 비용 절반 |
| 3차: 전략적 재미 | 게임적 사고 (Gamification) | "최저 고유가를 맞추는 전략" → 게임 요소 |
| 4차: 경쟁심 | 사회적 비교 (Social Comparison) | "현재 입찰자 수: 892명" 실시간 표시 |

### 2-4. 미니게임: "공짜로 벌어서 자연스럽게 쓰기" 소비 전환 브릿지

```
  [미니게임 플레이]        [포인트 획득]         [게임 참여 전환]
   퍼즐, 룰렛, 퀴즈  ──→   +3~10P 누적   ──→  "100P 모였네,
   완전 무료                                     DAILY 참여해볼까?"
                                                      │
                                                      ▼
                                               [자연스러운 소비]
                                               "내가 번 거니까
                                                부담 없이 써보자"
```

**핵심 설계**: 미니게임에서 번 포인트는 "내 노력의 대가"로 인식됩니다. 이는 충전한 포인트보다 **심리적 지출 허들이 낮습니다** (Mental Accounting - 하우스 머니 효과).

| 미니게임 포인트 축적량 | 자연스러운 전환 메시지 | 타겟 게임 |
|----------------------|---------------------|----------|
| 10P 도달 | "소문내기 1회 참여 가능!" | WISH (10P) |
| 50P 도달 | "DAILY 무료 참여 + 50P 남음" | DAILY |
| 100P 도달 | "DAILY 추가 참여 가능!" | DAILY (100P) |
| 200P+ 도달 | "PRIME 입찰 도전해보세요" | PRIME |

## 3. 선순환 강화 메커니즘

### 3-1. 소멸 압박 → 소비 유도

```
  [무료 포인트 획득]
        │
        ▼
  [90일 카운트다운 시작]
        │
        ├── 60일 경과: 잔액 옆 "30일 남음" 표시 (시각적 넛지)
        │
        ├── 83일 경과: 푸시 알림 "7일 후 ○○P 소멸 예정"
        │
        ├── 87일 경과: 인앱 배너 "3일 후 소멸! 지금 사용하세요"
        │
        └── 90일: 소멸 → "아까워!" 감정 → 다음부터 빨리 쓰는 습관 형성
```

<Callout type="warning">
  소멸 압박은 **유료 충전 포인트에는 적용되지 않습니다**. 무료 포인트에만 90일 소멸을 적용하여, 소비 압박과 과금 유저 보호를 동시에 달성합니다.
</Callout>

### 3-2. 연속 참여 보너스 → 이탈 비용 증가

연속 참여 기록은 **끊어지면 0부터 다시 시작**합니다. 이는 "여기서 멈추면 지금까지 쌓은 게 날아간다"는 매몰 비용을 만듭니다.

| 연속 일수 | 누적 보너스 | 이탈 시 손실 | 이탈 비용 체감 |
|----------|-----------|------------|--------------|
| 3일 | +20P | 20P | "별로 아깝지 않음" |
| 7일 | +70P | 70P | "좀 아깝네" |
| 14일 | +170P | 170P | "많이 아깝다" |
| 28일 | +670P | 670P | "절대 끊을 수 없다" |

**핵심**: 14일 이상 연속 참여한 유저의 이탈률은 초기 유저 대비 **1/5 이하**로 예상됩니다 (업계 벤치마크 기준).

### 3-3. 당첨 후기 → 소셜 프루프 → 신규 유입

```
  [당첨 발생] ──→ [후기 작성 유도] ──→ [후기 노출]
                   +30~100P 보상           │
                                    ┌──────┼──────┐
                                    ▼      ▼      ▼
                                  앱 내   SNS    커뮤니티
                                  피드    공유    공유
                                    │      │      │
                                    └──────┼──────┘
                                           ▼
                                    [소셜 프루프 형성]
                                    "진짜 당첨되네!"
                                           │
                                    ┌──────┼──────┐
                                    ▼      ▼      ▼
                                  기존    이탈    신규
                                  유저    유저    유저
                                  재참여  복귀    가입
```

| 후기 유형 | 보상 | 바이럴 계수 (예상) | 전략적 가치 |
|----------|------|------------------|------------|
| 텍스트 후기 | +30P | 0.05 | 앱 내 콘텐츠 |
| 사진 후기 | +50P | 0.15 | SNS 공유 적합 |
| 영상 후기 | +100P | 0.40 | 높은 바이럴 잠재력 |
| 인기 후기 (좋아요 50+) | +200P | 1.0+ | 최상위 바이럴 |

### 3-4. 소원 공감 → 게임 개설 → 참여자 증가 → 당첨 → 바이럴

WISH는 자체적으로 **완결된 미니 플라이휠**을 가집니다:

```
  ┌────────────────────────────────────────────┐
  │                                            │
  ▼                                            │
[소원 등록] ──→ [공감 수집] ──→ [게임 개설]      │
  +50P           +5P/건        목표 달성 시      │
                                    │            │
                                    ▼            │
                              [참여자 유입]       │
                              10P or 광고         │
                                    │            │
                                    ▼            │
                              [당첨 발생]         │
                              블록체인 추첨        │
                                    │            │
                              ┌─────┴─────┐      │
                              ▼           ▼      │
                          [당첨자]     [등록자]    │
                          상품 수령   1+1 혜택     │
                              │           │      │
                              ▼           ▼      │
                          [후기/공유] ──→ [신규 유입] ─┘
```

## 4. 소비 → 재획득 브릿지 설계

게임에서 포인트를 쓴 직후가 가장 중요한 순간입니다. **"다 써버렸다"는 허탈감을 "다시 모을 수 있다"는 희망으로 전환**해야 합니다.

### 4-1. 미당첨 위로 포인트 → 즉시 재참여

| 게임 | 미당첨 위로 | 재참여 가능 횟수 환산 | 심리적 효과 |
|------|-----------|---------------------|------------|
| DAILY | +3P | 0.03회 (누적형) | "모이면 다시 참여 가능" |
| WISH | +2P | 0.2회 | "5번 미당첨 = 1회 무료" |
| PRIME | +입찰금 50% | 즉시 절반 복구 | "절반은 남았으니 다시" |

### 4-2. 광고 시청 → 무료 참여 전환

```
  [포인트 부족!]
        │
        ▼
  ┌───────────────────────────────┐
  │  "광고 시청하고 무료로          │
  │   참여하시겠습니까?"            │
  │                               │
  │   [광고 시청하기]  [포인트 충전] │
  └───────────────────────────────┘
        │                  │
        ▼                  ▼
  광고 수익 발생       과금 전환
  + 유저 체류          매출 발생
```

| 광고 무료 참여 | 일일 한도 | 광고 단가 (예상) | 일일 수익 |
|--------------|----------|----------------|----------|
| DAILY 추가 참여 | 2회/일 | ₩300~500 | ₩600~1,000 |
| WISH 소문내기 | 3회/일 | ₩200~400 | ₩600~1,200 |
| **합계** | **5회/일** | | **₩1,200~2,200** |

### 4-3. 미니게임 포인트 보충 → 체류 시간 증가

```
  [게임 참여 후 잔액 부족]
        │
        ▼
  "미니게임으로 포인트를 모아보세요!"
        │
        ▼
  [미니게임 플레이] ──→ [+3~10P/판] ──→ [포인트 축적]
        │                                      │
        ▼                                      ▼
  체류 시간 증가                          재참여 가능
  (광고 노출 기회↑)                       (소비 루프 재진입)
```

**체류 시간 → 수익 전환 시뮬레이션**:

| 미니게임 플레이 시간 | 예상 광고 노출 | 광고 수익 | 획득 포인트 |
|--------------------|--------------|----------|-----------|
| 5분 (3판) | 1회 | ₩200 | +15P |
| 15분 (10판) | 3회 | ₩600 | +50P |
| 30분 (20판) | 5회 | ₩1,000 | +100P (일일 상한) |

## 5. 포인트 소비 심리학 (행동경제학 기반)

### 5-1. 앵커링 (Anchoring Effect)

유저가 포인트 가치를 판단할 때, **상품 정가를 앵커**로 설정합니다.

| 상품 | 정가 (앵커) | 참여비 | 할인율 체감 | 심리적 반응 |
|------|-----------|--------|-----------|------------|
| AirPods Pro | ₩359,000 | 100P (≈₩100) | 99.97% | "거의 공짜 도전" |
| iPhone 16 Pro | ₩1,890,000 | 500P (≈₩500) | 99.97% | "500원에 아이폰?" |
| 스타벅스 기프티콘 | ₩6,000 | 30P (≈₩30) | 99.50% | "커피 한 잔이 30원?" |

<Callout type="info">
  앵커링은 상품 상세 페이지에서 **정가를 크게, 참여비를 작게** 배치하여 극대화합니다. 할인율이 아닌 **절대 가격 차이**를 강조하는 것이 효과적입니다.
</Callout>

### 5-2. 매몰 비용 효과 (Sunk Cost Fallacy)

한 번 시간과 포인트를 투자하면, "여기서 그만두면 지금까지 한 게 아깝다"는 심리가 작동합니다.

| 매몰 비용 요소 | 축적 방식 | 이탈 시 손실 | 활용 설계 |
|--------------|----------|------------|----------|
| 연속 참여 기록 | 매일 1회 이상 참여 | 기록 초기화 | 연속 일수 UI + 보너스 |
| 소원 공감 수 | 공감 수집 활동 | 게임 미개설 | 공감 달성률 프로그레스 바 |
| PRIME 입찰 이력 | 입찰 참여 | 50% 환급만 | "이전 입찰 패턴 분석" 제공 |
| 미니게임 랭킹 | 플레이 누적 | 랭킹 하락 | 주간 랭킹 보상 |

### 5-3. 사회적 증거 (Social Proof)

다른 유저의 행동이 내 행동을 정당화합니다.

```
  ┌─────────────────────────────────────────┐
  │          실시간 소셜 프루프 요소           │
  ├─────────────────────────────────────────┤
  │                                         │
  │  🔴 LIVE  "방금 김○○님 에어팟 당첨!"     │
  │                                         │
  │  👥 "현재 1,234명 참여 중"               │
  │                                         │
  │  ⭐ "이 게임 평균 당첨률 3.2%"            │
  │                                         │
  │  💬 "당첨 후기 342개"                     │
  │                                         │
  └─────────────────────────────────────────┘
```

| 소셜 프루프 유형 | 노출 위치 | 예상 전환율 영향 |
|----------------|---------|----------------|
| 실시간 당첨 알림 | 홈 피드 상단 | +15~25% 참여율 |
| 참여자 수 카운터 | 게임 상세 페이지 | +10~20% 참여율 |
| 당첨 후기 썸네일 | 게임 하단 갤러리 | +20~30% 참여율 |
| 친구 참여 알림 | 푸시 알림 | +25~35% 복귀율 |

### 5-4. 희소성 (Scarcity)

"지금 아니면 놓친다"는 긴급성이 소비를 가속합니다.

| 희소성 유형 | 적용 게임 | 구체적 설계 |
|------------|---------|------------|
| **시간 한정** | DAILY | "오늘 23:59까지" 마감 타이머 |
| **수량 한정** | DAILY, WISH | "남은 수량: 3개" |
| **참여자 한정** | WISH | "최대 500명, 현재 487명" |
| **시즌 한정** | 전체 | "봄맞이 한정 상품" (기간제) |

### 5-5. 감정적 소유권 (Endowment Effect)

"내 것"이라고 느끼면 가치를 더 높게 평가합니다.

| 감정적 소유권 요소 | 적용 영역 | 설계 방법 |
|------------------|---------|----------|
| "내 소원" | WISH | 소원 등록 시 "내 소원함" 탭에 저장 |
| "내 입찰" | PRIME | 입찰 후 "내 입찰 현황" 실시간 추적 |
| "내 연속 기록" | DAILY | 프로필에 "연속 ○일차" 배지 표시 |
| "내 포인트" | 전체 | 잔액을 "내 포인트"로 명명, 증감 애니메이션 |
