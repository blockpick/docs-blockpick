## API 캐싱

### Redis 기반 서버 캐싱

| API | TTL | 캐시 키 | 무효화 조건 |
|-----|-----|---------|------------|
| **게임 목록** | 30초 | `games:list:{type}:{page}` | 새 게임 생성/종료 시 |
| **소원 목록** | 60초 | `wishes:list:{sort}:{page}` | 새 소원 등록/공감 시 |
| **유저 프로필** | 5분 | `user:{userId}:profile` | 프로필 수정 시 |
| **앱 설정** | 10분 | `config:app` | 설정 변경 시 |
| **게임 상세** | **캐시 안 함** | - | 실시간 참여 현황 반영 필요 |
| **게임 결과** | 1시간 | `game:{gameId}:result` | 결과 확정 후 변경 없음 |

<Callout type="warning">
  게임 상세 페이지는 실시간 참여 현황(남은 블록 수, 참여자 수)을 표시해야 하므로 **캐시를 사용하지 않습니다**. 대신 WebSocket/SSE로 실시간 업데이트를 처리합니다.
</Callout>

### 캐시 무효화 전략

- **이벤트 기반**: 데이터 변경 이벤트 발생 시 관련 캐시 키 삭제
- **TTL 기반**: 캐시 만료 시 자동 갱신 (Cache-Aside 패턴)
- **버전 태그**: 캐시 키에 버전 포함하여 배포 시 일괄 무효화

## 이미지 CDN

### AWS CloudFront 구성

| 항목 | 설정 |
|------|------|
| **Origin** | S3 버킷 (소원 상품 이미지, 유저 프로필 이미지) |
| **CDN** | AWS CloudFront (글로벌 엣지) |
| **SSL** | HTTPS 강제 |
| **캐시 TTL** | 이미지: 30일, HTML/CSS/JS: 1일 |

### 이미지 리사이징

| 용도 | 크기 | 포맷 | 사용처 |
|------|------|------|--------|
| **썸네일** | 200 x 200px | WebP (fallback: JPEG) | 소원 목록, 게임 목록 |
| **상세** | 800 x 800px | WebP (fallback: JPEG) | 소원 상세, 게임 상세 |
| **원본** | 원본 크기 | 원본 포맷 | 관리자 검수용 |

- **리사이징 방식**: Lambda@Edge 또는 CloudFront Functions로 온디맨드 리사이징
- **URL 패턴**: `https://cdn.blockpick.io/images/{size}/{imageId}.webp`

## GraphQL 최적화

### DataLoader (N+1 문제 해결)

```
# Before: N+1 쿼리
게임 목록 조회 (1 쿼리)
  └→ 각 게임의 생성자 정보 (N 쿼리) ← 문제!

# After: DataLoader 적용
게임 목록 조회 (1 쿼리)
  └→ 생성자 정보 일괄 조회 (1 쿼리) ← 해결
```

### 쿼리 복잡도 제한

| 제한 | 값 | 사유 |
|------|-----|------|
| **쿼리 깊이 (depth)** | 최대 5 | 중첩 관계 과도 조회 방지 |
| **쿼리 복잡도 (complexity)** | 최대 100 | 과도한 필드 요청 방지 |
| **목록 최대 개수** | `first` / `last` 최대 50 | 대량 데이터 한 번에 조회 방지 |

<Callout type="info">
  복잡도 초과 시 `QUERY_TOO_COMPLEX` 에러를 반환하고, 클라이언트에 쿼리 분할을 안내합니다.
</Callout>

### Persisted Queries

- 프로덕션에서는 **Persisted Queries**를 사용하여 쿼리 문자열 전송을 줄이고, 보안성을 높입니다.
- 쿼리 해시(SHA-256)로 요청하며, 등록되지 않은 쿼리는 차단합니다.

## 앱 로컬 캐시

### Riverpod 기반 인메모리 캐시

| 데이터 | 캐시 전략 | 갱신 조건 |
|--------|-----------|-----------|
| 게임 목록 | `staleTime: 30초` | 탭 전환 시 백그라운드 리패치 |
| 소원 목록 | `staleTime: 60초` | Pull-to-refresh 시 즉시 갱신 |
| 유저 프로필 | `staleTime: 5분` | 프로필 수정 후 즉시 갱신 |
| 게임 상세 | `staleTime: 0` (항상 리패치) | WebSocket 실시간 업데이트 |

### 로컬 영속 저장소

| 저장소 | 용도 | 데이터 |
|--------|------|--------|
| **SharedPreferences** | 간단한 설정값 | 알림 설정, DND 시간, 테마 |
| **Hive** | 구조화된 로컬 데이터 | 최근 참여 게임 목록, 캐시된 유저 정보 |
| **SQLite (drift)** | 복잡한 쿼리 필요 시 | 오프라인 모드 지원 데이터 (Phase 2) |

## 네트워크 최적화

| 전략 | 내용 |
|------|------|
| **HTTP/2** | API 서버 HTTP/2 적용 (멀티플렉싱) |
| **gzip 압축** | API 응답 gzip 압축 (텍스트 응답 70% 이상 절감) |
| **페이지네이션** | 커서 기반 페이지네이션 (오프셋 기반보다 성능 우수) |
| **선택적 필드** | GraphQL의 필드 선택으로 필요한 데이터만 전송 |
| **이미지 Lazy Loading** | 화면에 보이는 이미지만 우선 로드 |
