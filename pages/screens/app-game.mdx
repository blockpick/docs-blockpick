## 게임 디스패처 (A-30)

`GameDispatcherScreen`은 게임 ID를 받아 타입에 따라 적절한 게임 화면으로 라우팅합니다.

- DAILY → 블록픽 판 (블록체인 기반 보드, 고정 상품)
- VIBE → 블록픽 판 (블록체인 기반 보드, 소원 상품)
- PRIME → 역경매 입찰 화면 (스크롤 리스트 UI)

## 블록픽 판 (A-31, A-32)

블록체인 기반 보드 게임 방식:
1. **블록 선택**: 보드 위 블록을 선택
2. **블록체인 기록**: 선택 정보가 블록체인에 암호화·기록 (AES-256 암호화, SHA-256 해시)
3. **결과 확인**: 게임 종료 시 블록체인에서 당첨 여부 판정 (투명성 보장)

### DAILY vs VIBE 차이

| 항목 | DAILY | VIBE |
|------|-------|------|
| 상품 | 운영 등록 상품 (커피, 라면 등) | 사용자 등록 소원 상품 |
| 참여비 | 고정 | 소문내기 10원 |
| 블록체인 | 좌표 암호화 + 온체인 기록 | 좌표 암호화 + 온체인 기록 |

## 가챠 게임 → 미니게임으로 이동

기존 가챠(2단계 좌표 선택) 방식은 미니게임 섹션으로 이동되었습니다.
자세한 내용은 [미니게임 (30종+)](/game-system/mini-games)을 참조하세요.

## 그리드 게임 (홀딩)

SELECT 타입 그리드 게임은 현재 홀딩 상태입니다. 구현은 완료되었으나 서비스에 포함하지 않습니다.
