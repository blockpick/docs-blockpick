# 사용 시나리오

주요 페르소나별 사용 시나리오입니다.

## 시나리오 1: 일일 게임 참여

> 김민수 (27세, 직장인) - 매일 출석체크, DAILY 게임 참여

1. 김민수가 출근길에 앱을 열면 홈 화면이 표시된다
2. "오늘의 DAILY 게임" 배너를 탭한다
3. 게임 상세 화면에서 상품 정보와 남은 시간을 확인한다
4. "참여하기" 버튼을 탭한다
5. 가챠 화면에서 X좌표를 선택한다 (애니메이션 정지)
6. Y좌표를 선택한다
7. 선택한 좌표가 블록체인에 기록된다
8. 게임 종료 시간에 결과를 확인한다

## 시나리오 2: 파트너 게임 비회원 참여

> 박지현 (45세, 주부) - 간단한 참여 선호

1. 박지현이 SNS에서 파트너 게임 링크를 클릭한다
2. 웹 브라우저에서 파트너 게임 랜딩 페이지가 열린다
3. 전화번호를 입력한다
4. 가챠 게임에 참여한다
5. 즉시 당첨 여부가 표시된다
6. 당첨 시 기프티쇼 쿠폰 URL이 문자로 발송된다

## 시나리오 3: 크리에이티브 픽 모드 사용

> 이수진 (33세, 프리랜서) - 재미있는 게임 경험 추구

1. 이수진이 이벤트 탭에서 SELECT 게임을 선택한다
2. 게임 상세에서 "더 많은 픽 방식" 메뉴를 탭한다
3. "Gravity" 모드를 선택한다
4. 디바이스를 기울여서 좌표를 결정한다
5. 확정된 좌표로 게임에 참여한다

## 시나리오 4: 당첨 후 상품 수령

1. 김민수가 알림을 통해 당첨 소식을 확인한다
2. 마이페이지 > 당첨 이력에서 당첨 상품을 확인한다
3. 배송지를 입력/확인한다
4. 블록체인 트랜잭션 해시로 결과의 공정성을 검증한다
5. 상품이 배송되면 후기를 작성한다

## 페르소나 요약

### 김민수 (27세, 직장인)

| 항목 | 내용 |
|------|------|
| **특성** | 모바일 게임에 익숙, 출퇴근 시간에 앱 사용 |
| **목표** | 소액으로 고가 상품에 당첨되고 싶음 |
| **니즈** | 빠른 참여 플로우, 결과의 공정성 확인, 다양한 게임 모드 |
| **Pain Point** | 기존 추첨 서비스의 불투명한 결과, 반복적이고 지루한 참여 방식 |

### 이수진 (33세, 프리랜서)

| 항목 | 내용 |
|------|------|
| **특성** | SNS 활동 활발, 새로운 앱 탐색을 즐김 |
| **목표** | 재미있는 게임 경험, 당첨 후기 공유 |
| **니즈** | 시각적으로 화려한 게임, 소셜 공유 기능 |
| **Pain Point** | 단순 클릭만 반복하는 추첨의 지루함 |

### 박지현 (45세, 주부)

| 항목 | 내용 |
|------|------|
| **특성** | 모바일 앱 사용에 익숙하지 않음, 실용적 상품에 관심 |
| **목표** | 생활용품/식품 등 실용적 상품 당첨 |
| **니즈** | 직관적인 UI, 쉬운 참여 방법, 배송 추적 |
| **Pain Point** | 복잡한 가입 절차, 이해하기 어려운 블록체인 개념 |
